# Remotion Lyric Video 최종 개발 문서 및 개선 프롬프트

이 문서는 현재 `lyric-video-remotion` 프로젝트의 최종 구현 상태를 설명하고, 이후 Claude Code 또는 다른 코드 에이전트가 추가 개선을 진행할 때 기준 프롬프트로 사용하기 위한 문서이다.

## 1. 프로젝트 목적

이 프로젝트는 로컬 미디어 파일을 이용해 유튜브 업로드용 16:9 MP4 가사 영상을 생성하는 Remotion + TypeScript 프로젝트이다.

핵심 목표:

- `public/media` 폴더의 로컬 오디오 사용
- LRC 또는 SRT 자막 파일을 파싱해 가사 싱크 표시
- 정지 이미지 또는 MP4 배경 자동 적용
- 하단 중앙에 간결한 흰색 가사 표시
- 가사 아래에 작고 차분한 웨이브 파형 표시
- Remotion Studio 미리보기와 최종 MP4 렌더 모두 지원

## 2. 현재 기술 스택

- Remotion `4.0.458`
- React `19.2.0`
- TypeScript `5.9.3`
- Vite `7.1.12`
- Windows 배치 파일 실행 지원

주요 npm scripts:

```bash
npm run dev
npm run build
npm run typecheck
```

## 4. 미디어 파일 규칙

모든 사용자 미디어는 아래 폴더에 둔다.

```text
public/media/
```

### 4.1 오디오

지원:

```text
audio.mp3
audio.wav
```

우선순위:

```text
1. /media/audio.mp3
2. /media/audio.wav
```

현재 `Root.tsx`의 `AUDIO_CANDIDATES`에서 위 순서로 선택한다. 파일명이 반드시 `audio.mp3` 또는 `audio.wav`여야 한다.

### 4.2 가사

지원:

```text
lyrics.lrc
lyrics.srt
```

우선순위:

```text
1. lyrics.lrc
2. lyrics.srt
```

LRC가 존재하고 유효한 라인이 있으면 SRT보다 우선한다.

### 4.3 배경

지원:

```text
mp4
jpg
jpeg
png
```

우선순위:

```text
1. background.mp4 또는 다른 mp4 파일
2. background.jpg / background.jpeg / background.png 또는 다른 jpg/jpeg/png 파일
3. 기본 애니메이션 배경
```

배경 파일은 `scripts/write-media-manifest.mjs`가 자동 스캔해 `src/mediaManifest.ts`를 생성 또는 갱신한다.

배경 선택 점수:

- `background.*` 최우선
- `bg.*`
- 이름에 `background` 포함
- 이름에 `bg` 포함
- 나머지는 파일명 정렬

움직이는 배경은 GIF보다 MP4를 권장한다. 배경 MP4는 무음 영상이 가장 좋다. 현재 코드는 배경 MP4를 `muted`, `volume={0}`, `loop`로 렌더링한다.

권장 해상도:

```text
1920 x 1080
```

다른 비율도 `objectFit: 'cover'`로 화면을 채우지만 일부가 잘릴 수 있다.

## 6. Remotion 구성

`Root.tsx`:

- 오디오 파일 존재 여부 확인
- LRC/SRT 텍스트 fetch
- `@remotion/media-utils`의 `getAudioDurationInSeconds()`로 오디오 길이 감지
- 가사 파싱 결과와 오디오 길이 중 더 긴 값을 기준으로 영상 duration 계산
- Composition 설정

현재 Composition:

```text
id: lyric-video
width: 1920
height: 1080
fps: 30
```

`remotion.config.ts`:

```ts
Config.setVideoImageFormat('jpeg');
Config.setOverwriteOutput(true);
```

## 7. 가사 파서 동작

`src/parsers.ts` 담당.

### 7.1 LRC

지원:

- `[mm:ss.xx]`
- `[mm:ss.xxx]`
- `[mm:ss]`
- 한 줄에 여러 timestamp
- UTF-8 한국어/일본어/영어 혼합 텍스트

메타 태그와 timestamp는 제거하고 실제 가사만 추출한다.

### 7.2 SRT

지원:

- `HH:MM:SS,mmm --> HH:MM:SS,mmm`
- 여러 줄 자막 병합
- UTF-8 텍스트

### 7.3 Duration

파서 fallback:

- 기본 라인 길이: `3.2초`
- outro: `4초`
- 최소 duration: `10초`

Root에서는 오디오 길이를 감지할 수 있으면 `audioDurationSeconds + 1`도 고려한다.

## 8. 화면 디자인 현재 상태

`src/LyricVideo.tsx` 담당.

### 8.1 배경

배경 종류:

- `video`: `Video` 컴포넌트 사용, `objectFit: cover`, 무음, 반복
- `image`: `Img` 컴포넌트 사용, `objectFit: cover`, 미세 zoom 애니메이션
- `none`: 코드 기반 어두운 애니메이션 배경

모든 배경 위에는 자막 가독성을 위해 어두운 gradient/radial overlay가 올라간다.

### 8.2 가사

현재 가사는 하단 중앙에 하나의 active line만 표시한다.

주요 스타일:

- 위치: 화면 하단 약 `77%`
- 색상: 현재 라인 `#ffffff`
- 이웃 라인은 현재 구조상 사실상 표시하지 않음
- 폰트: `Inter`, `Pretendard`, `Noto Sans KR`, `Noto Sans JP`, `Malgun Gothic`, system fallback
- 크기: `clamp(24px, 2.8vw, 44px)`
- 흰색 + 약한 shadow
- `overflowWrap: anywhere`, `textWrap: balance`

사용자 의도:

- 유튜브에 별도 가사 정리를 올릴 예정이므로 영상 자막은 너무 크지 않게
- 가독성 좋고 피로감 적게
- 화려한 색상보다 흰색 중심

### 8.3 웨이브 파형

가사 아래 중앙에 작은 SVG 파형을 표시한다.

현재 특징:

- 폭 `260px`
- 높이 `86px`
- 위치: 화면 하단 약 `82.7%`
- 색상: 차분한 cyan/teal 계열
- 너무 크거나 복잡하지 않게 조절된 상태

## 11. 현재 알려진 제한사항

- 오디오 파일명은 자동 탐색이 아니라 `audio.mp3` 또는 `audio.wav` 고정이다.
- 가사 파일명도 `lyrics.lrc` 또는 `lyrics.srt` 고정이다.
- 배경은 media 폴더 자동 탐색이지만 GIF는 현재 제외되어 있다. 움직이는 배경은 MP4 권장.
- 배경 MP4는 반복 재생되지만, 배경 영상의 duration을 오디오에 맞게 별도 타임 스트레칭하지 않는다.
- 웨이브 파형은 실제 오디오 분석 기반이 아니라 deterministic SVG 애니메이션이다.
- 현재 자막은 active line 하나만 보이는 방향으로 조정되어 있다. 원래 요구사항의 previous/current/next 표시보다 더 간결한 최종 사용자 취향을 반영한 상태이다.

## 12. 향후 개선 후보

우선순위가 높은 개선:

1. `public/media`에서 오디오와 가사도 자동 탐색
2. 선택된 파일명을 화면 또는 콘솔에 명확히 표시
3. 실제 오디오 waveform 분석 기반 파형 생성
4. 배경 MP4 길이가 짧을 때 loop 경계가 티나지 않도록 crossfade 또는 blur transition
5. `render-video.bat`에서 배경 파일 감지 결과 출력
6. render 품질 옵션 추가: `--crf`, codec, audio bitrate
7. YouTube Shorts용 9:16 composition 옵션 추가
8. 긴 가사 라인 자동 줄바꿈과 폰트 사이즈 세부 튜닝

주의해서 접근할 개선:

- 자막을 다시 화려한 색으로 만들지 말 것. 사용자는 흰색 자막을 선호한다.
- 파형을 너무 크게 만들지 말 것. 사용자는 가사 아래 작고 간결한 파형을 선호한다.
- 배경이 너무 밝으면 자막 가독성이 떨어진다. overlay는 유지하는 편이 좋다.
- 웹페이지 미리보기와 최종 render의 차이를 혼동하지 않도록 문서와 배치 파일 메시지를 명확히 유지할 것.

## 13. Claude Code용 최종 개선 프롬프트

아래 프롬프트를 Claude Code에 그대로 전달해 추가 개선을 진행할 수 있다.

```text
You are a senior Remotion + TypeScript + Windows automation engineer.

Work in the existing project:
lyric-video-remotion

Read FINAL_DEVELOPMENT_PROMPT.md first and treat it as the current product specification. Then inspect the actual source files before editing:

- package.json
- src/Root.tsx
- src/LyricVideo.tsx
- src/parsers.ts
- src/mediaManifest.ts
- scripts/write-media-manifest.mjs
- start-studio.bat
- render-video.bat

Project goal:
Maintain and improve a production-runnable Remotion lyric video generator for YouTube 16:9 MP4 output.

Current intended behavior:
- Load local audio from public/media/audio.mp3 or public/media/audio.wav.
- Parse timed lyrics from public/media/lyrics.lrc or public/media/lyrics.srt.
- Prefer LRC over SRT.
- Detect optional background media from public/media.
- Prefer background MP4 over image.
- Support jpg, jpeg, png image backgrounds.
- If no background exists, use the built-in animated dark background.
- Render final video to out/lyric-video.mp4.
- Keep the composition 1920x1080, 30fps.
- Keep subtitle style minimal, white, readable, and not too large.
- Keep the waveform small, centered under the lyric, and visually calm.

Important user preferences:
- The final result must be a file suitable for YouTube upload.
- Remotion Studio is only preview; npm run build or render-video.bat creates the actual MP4.
- Do not add a visible title such as "Lyric Video" to the upper-left corner.
- Do not make the waveform large, colorful, or visually tiring.
- Do not make the subtitles huge or colorful. White subtitle text is preferred.
- Background images/videos are optional and should only be used when present.
- Background media should be 16:9, preferably 1920x1080.
- If background is bright, keep overlays so lyrics remain readable.

Implementation constraints:
- Use Remotion primitives: staticFile(), Audio, Video, Img, useCurrentFrame(), useVideoConfig().
- Do not use timers or intervals for lyric sync.
- Keep TypeScript strict and avoid implicit any.
- Preserve UTF-8 support for Korean, Japanese, English, and mixed lyrics.
- Keep output reproducible on Windows.
- Keep batch files user-friendly.
- Do not remove existing media files or generated output unless explicitly asked.

Before finishing any change:
1. Run npm run typecheck.
2. Run npx remotion still src/index.ts lyric-video out/preview-check.png --frame=150.
3. If the change affects final render behavior, run npm run build.
4. Report what changed, commands run, and where the final MP4 is located.

Potential next improvement task:
Improve media discovery so audio and lyric files can be automatically selected from public/media without requiring exact names, while preserving the current explicit audio.mp3/audio.wav and lyrics.lrc/lyrics.srt priority. If multiple candidates exist, choose the most appropriate file and print the selection in render-video.bat output.
```
