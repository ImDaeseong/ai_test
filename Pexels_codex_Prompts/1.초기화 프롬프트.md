# Codex Prompt: Gemini + Pexels + FFmpeg 기반 유튜브 쇼츠/뮤직비디오 자동 생성기

## 역할

당신은 Senior Software Architect이자 Full-Stack/Automation Engineer이다.

목표는 로컬 `data/` 폴더에 있는 가사/자막/음악 파일을 기반으로 Gemini가 영상 장면을 설계하고, Pexels API에서 장면별 스톡 영상을 검색/다운로드한 뒤, FFmpeg로 유튜브 업로드 친화적인 세로형 MP4를 자동 생성하는 프로젝트를 구현하는 것이다.

최종 사용자는 복잡한 명령어를 몰라도 `run.bat`만 실행하면 된다.

```bat
run.bat
```

실행 결과:

```text
output/final_data.mp4
output/index.html
```

`output/index.html`은 브라우저에서 최종 영상, 장면별 미리보기 영상, 한글 장면 설명, 검색 키워드, Pexels ID 등을 확인할 수 있는 정적 HTML 리포트다.

```text

  ↓

  ↓

  ↓

  ↓

  ↓
```

## 중요한 구현 원칙

1. API 키는 반드시 `.env`로 관리하고 코드에 하드코딩하지 말 것.
2. `.env.example`에는 예시 값만 넣을 것.
3. FastAPI나 서버는 사용하지 말 것. 브라우저 확인은 정적 HTML 파일로 처리할 것.
4. 최종 사용 흐름은 `run.bat` 하나로 끝나야 한다.
5. 개발자용 메뉴는 `dev_tools.bat`에 분리할 것.
6. 최종 MP4 길이는 음악 파일 길이와 같아야 한다.
7. WAV 파일을 반드시 지원할 것.
8. 최종 MP4는 유튜브 업로드 친화 형식이어야 한다.
9. Pexels 원본 영상과 처리 파일이 누적되므로 정리 도구를 제공할 것.
10. FFmpeg 호출은 `subprocess.run([...])` 형태로 실행하고 shell string을 쓰지 말 것.
11. 외부 API 호출에는 timeout, retry, 예외 처리를 넣을 것.
12. Gemini 응답은 반드시 JSON 파싱 및 Pydantic 검증을 거칠 것.

## 데이터 입력 요구사항

`data/` 폴더에서 다음 파일을 자동 감지한다.

### 텍스트/가사

우선순위:

1. `.lrc`
2. `.srt`
3. `.txt`

`.lrc`는 `[00:00.00]` 같은 timestamp와 `[Verse]`, `[Intro]` 같은 태그를 제거하고 순수 가사만 추출한다.

`.srt`는 cue 번호와 timestamp를 제거하고 대사만 추출한다.

### 음악

지원 확장자:

```text
.wav
.mp3
.m4a
.aac
```

WAV는 반드시 지원해야 한다.

### 자막

`.srt` 파일이 있으면 최종 MP4에 `mov_text` subtitle track으로 mux한다.

### 이미지

`.png`, `.jpg`, `.jpeg`, `.webp`는 감지하되 초기 버전에서는 렌더링에 사용하지 않아도 된다. HTML 리포트에는 감지된 파일 경로를 표시한다.

## Gemini 장면 생성 요구사항

Gemini는 입력 텍스트를 기반으로 장면 JSON 배열을 반환해야 한다.

프롬프트 요구사항:

```text
You are an expert music video director and visual scene planner.

Analyze the following text and split it into cinematic video scenes.

Return ONLY valid JSON array.
Do not include markdown.
Do not include explanation.

Create 6 to 8 scenes unless the input is extremely short.
Keep each initial duration between 4 and 8 seconds.
The application will retime scenes later to match the music duration, so focus on clear visual variety.

Each scene must include:
- scene: detailed visual scene description in English
- scene_ko: natural Korean explanation of the visual scene
- search_keywords: short Pexels search keywords in English
- mood: emotional mood
- camera: camera direction
- orientation: portrait, landscape, or square
- duration: duration in seconds

Preferred orientation: {orientation}
Visual style: {style}

Input text:
{text}

Output format:
[
  {
    "scene": "...",
    "scene_ko": "...",
    "search_keywords": "...",
    "mood": "...",
    "camera": "...",
    "orientation": "portrait",
    "duration": 5
  }
]
```

Pydantic 모델:

```text
scene: non-empty string
scene_ko: optional string
search_keywords: non-empty string
mood: non-empty string
camera: non-empty string
orientation: portrait | landscape | square
duration: number, > 0, <= 60
```

Gemini 응답이 JSON이 아니거나 빈 배열이면 예외 처리할 것.

## Pexels 서비스 요구사항

Pexels API:

```http
GET https://api.pexels.com/videos/search
```

요구사항:

- `PEXELS_API_KEY`가 없으면 명확한 오류
- 검색어 기반 영상 검색
- orientation 파라미터 사용
- 검색 결과는 `storage/cache/`에 JSON 캐시
- 결과가 없으면 fallback keyword로 재검색
- 영상 선택 기준:
  - orientation 일치
  - 해상도
  - scene duration과 영상 길이 차이
- 다운로드 가능한 MP4 중 가장 해상도가 높은 파일 선택

다운로드 요구사항:

- `storage/raw_videos/`에 저장
- 같은 Pexels video id는 중복 다운로드하지 않음
- 파일명은 안전한 slug 처리
- `MAX_DOWNLOAD_MB`보다 큰 다운로드는 중단

## FFmpeg 요구사항

### 음악 길이 기준

음악 파일이 있으면:

1. `ffprobe`로 음악 duration 측정
2. Gemini scene duration 합계를 음악 duration에 맞게 비례 조정
3. 마지막 scene에서 drift 보정
4. 최종 MP4 duration은 음악 duration과 같아야 함

예:

```text
input music: 181.992 seconds
final mp4 : 181.992 seconds
```

### Pexels 영상이 scene 길이보다 짧은 경우

`-stream_loop -1`을 사용해 source clip을 반복하고, `-t scene_duration`으로 필요한 길이만큼 잘라낸다.

### 장면별 처리

각 scene마다:

```text
storage/processed/final_data_scene_001.mp4
storage/processed/final_data_scene_002.mp4
...
```

을 생성하고 HTML 리포트에 장면별 미리보기 영상으로 연결한다.

### 같은 output stem 정리

렌더링 시작 전 다음 파일을 자동 삭제:

```text
storage/processed/{output_stem}_scene_*.mp4
storage/processed/{output_stem}_silent.mp4
storage/processed/concat_{output_stem}*.txt
```

이렇게 해서 scene 개수가 줄어도 오래된 scene preview가 남지 않게 한다.

### 유튜브 친화 최종 MP4

최종 export는 다음 조건을 만족해야 한다.

영상:

```text
container: MP4
codec: H.264
profile: High
pix_fmt: yuv420p
fps: 30
resolution portrait: 1080x1920
resolution landscape: 1920x1080
resolution square: 1080x1080
GOP: closed GOP, 15 frames for 30fps
movflags: +faststart
```

오디오:

```text
codec: AAC-LC
sample_rate: 48000
channels: 2
bitrate: 320k
```

자막:

```text
codec: mov_text
```

FFmpeg 명령은 반드시 list argument로 구성하고 `shell=True`를 사용하지 않는다.

## run.bat 요구사항

일반 사용자는 `run.bat`만 실행하면 된다.

동작:

1. 현재 폴더로 이동
2. `PYTHONUTF8=1` 설정
3. `.venv` 없으면 생성
4. `requirements.txt` 설치
5. `.env` 없으면 `.env.example`에서 복사 후 사용자에게 편집 안내
6. FFmpeg 설치 확인
7. `data/` 폴더 확인
8. `output/` 폴더 없으면 생성
9. 테스트 실행
10. `scripts/generate_from_data.py` 실행
11. `output/final_data.mp4` 생성
12. `output/index.html` 생성
13. 브라우저로 리포트 열기

명령:

```bat
".venv\Scripts\python.exe" scripts\generate_from_data.py --data-dir data --output output\final_data.mp4 --report output\index.html --orientation portrait --open
```
