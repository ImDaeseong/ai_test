# 프로젝트 시작 기록

## 1. 기본 정보

- 프로젝트명: `ai-webtoon_capcut`
- 목적: 곡마다 다른 이미지 수·음원 길이·Suno 자막을 재사용 가능한 편집 타임라인으로 변환한다.
- 담당자: 개인 프로젝트 소유자
- 시작일: 2026-06-06
- 프로젝트 구분: 개인 프로젝트
- 예상 배포일: 보류 - Remotion 렌더와 5곡 end-to-end 검증 후 결정

## 2. 데이터 민감도

- 분류: 일반/내부
- 처리 데이터: 개인 창작 이미지, 음원, 가사, 스토리보드
- 저장 금지: 회사 자료, 고객 자료, 인증 정보, private URL

## 3. 보안 기반

- 외부 API: 없음
- 비밀값: 현재 없음
- `.env.example`: 외부 API를 사용하지 않음을 명시
- 원본 데이터: 읽기 전용
- 원본 입력: `input/{노래명}`에서만 읽음
- 산출물: `output/{노래명}/{run_id}`에만 생성
- 비밀값 검사: `scripts/validate-project.ps1`

## 4. 아키텍처 결정

- 언어: Python 3.11 이상
- 프레임워크: 없음, 표준 라이브러리 우선
- DB/인증: 없음
- 실행 환경: Windows 로컬 CLI
- 배포 환경: 미정

결정:

1. Python 분석 엔진과 영상 렌더러를 분리한다.
   - 이유: 분석·계획을 Remotion 설치와 독립적으로 검증하기 위해서다.
   - 포기한 대안: 처음부터 Node/Remotion 단일 구현
2. `input` 원본은 읽기 전용이고 결과는 `output`에 저장한다.
   - 이유: 반복 실행과 원본 복구 가능성을 보장하기 위해서다.

## 5. 관찰 가능성

- 로그: stderr JSON Lines
- 결과 보고서: `output/{노래명}/{run_id}/reports/qa.json`
- 오류 알림: 로컬 CLI 종료 코드와 구조화 오류
- 모니터링 대상: 실행 단계, 곡 식별자, 처리 결과, 오류 코드

## 6. AI 코딩 계획

- 사용 도구: Codex
- AI 담당: 설계 보강, 구현, 자동 검사
- 사람 담당: 영상 품질, 자막 체감 싱크, CapCut import, 공개/배포 승인
- AI 생성 코드는 `AI_CODING_REVIEW.md` 기준으로 재검수한다.

## 7. 롤백

- 코드: 이전 Git commit 또는 변경 전 백업으로 복원
- 데이터: 원본을 수정하지 않으므로 롤백 불필요
- 실행 결과: 해당 `output/{노래명}/{run_id}` 삭제 후 재실행
