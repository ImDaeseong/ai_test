# 보안 설계

## 적용 유형

- 유형: 개인 로컬 CLI
- 공개 범위: 현재 로컬, 향후 공개 가능성 검토
- 외부 연결: 없음
- 인증/DB: 없음

## 환경변수

현재 필수 환경변수와 비밀값은 없다. 향후 외부 API를 추가할 경우 실제 값은
환경변수 또는 로컬 `.env`로 주입하고 저장소에는 키 이름만 기록한다.

## 보안 경계

| 항목 | 상태 | 비고 |
|---|---|---|
| 자격증명 하드코딩 금지 | 적용 | 현재 자격증명 없음 |
| 원본 파일 불변 | 적용 | input/output 읽기 전용 |
| 산출물 경로 격리 | 적용 | workspace 내부 |
| 절대 사용자 경로 제거 | 적용 | manifest 상대 경로 |
| 입력 파일 형식 검사 | 부분 적용 | WAV, PNG/JPEG, 일부 WebP |
| 외부 프로세스 제한 | 적용 | 고정 프로젝트 경로, timeout, 종료 코드 검사 |
| 로그 민감정보 배제 | 적용 | 경로·토큰 원문 미기록 |

## STRIDE 요약

- Spoofing: 인증 기능이 없어 해당 없음
- Tampering: 원본 해시 기록과 읽기 전용 정책으로 탐지
- Repudiation: run manifest와 QA 보고서로 실행 근거 보존
- Information Disclosure: 비밀값 미사용, 절대 개인 경로 제거
- Denial of Service: 배치 곡별 실패 격리, 향후 파일 크기 제한 필요
- Elevation of Privilege: 관리자 권한과 네트워크 기능을 요구하지 않음

## 하지 않는 것

- `.env` 생성 또는 수정
- 외부 API 호출
- CapCut 비공개 포맷 직접 수정
- input/output 원본 덮어쓰기
- 신뢰도 낮은 자막 자동 승인

## 남은 보안 작업

- 비정상적으로 큰 파일에 대한 크기 제한
- 공개 전 Git history secret scan
