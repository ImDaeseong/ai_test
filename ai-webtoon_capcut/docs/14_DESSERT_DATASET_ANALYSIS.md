# 디저트 데이터셋 분석

## 1. 판정

- 검증 곡 순번: 2
- 데이터 상태: `REVIEW_REQUIRED`
- 영상 생성 가능 여부: 조건부 가능
- 자동 렌더링 투입 여부: 자막 및 패널 메타데이터 정규화 후 가능
- 원본 경로: `ai-webtoon/output/디저트`

`디저트` 폴더에는 이미지, 패널 프롬프트, WAV, LRC, SRT가 모두 존재한다.
따라서 자산 부족으로 차단되는 곡은 아니다. 다만 원본 메타데이터를 그대로
사용하면 자막 과다 노출과 섹션 오분류가 발생할 수 있으므로 원본을 수정하지
않고 파생 정규화 파일을 생성해야 한다.

## 3. 이미지 검사

- 44장: 1659x948
- `panel_029_verse_medium.png`: 1661x947
- 모든 파일은 읽을 수 있으며 번호는 정상이다.
- 대표 컷 검사 결과 네온 마젠타, 해골 밴드, 도시와 디저트 소재가 같은
  세계관으로 유지된다.

29번 이미지는 렌더 전에 목표 캔버스로 `cover` 또는 `contain + background`
처리해야 한다. 원본 파일을 강제로 리사이즈하지 말고 렌더러에서 통일한다.

## 4. 스토리보드 검사

스토리보드에는 총 45패널이 선언되어 있다.

| 구간 | 패널 |
|---|---:|
| Intro | 3 |
| Instrumental | 4 |
| Verse 1 | 5 |
| Chorus | 14 |
| Drop | 7 |
| Verse 2 | 5 |
| Bridge | 4 |
| Outro | 3 |

원본 권장 지속 시간을 합산하면 231초로 음원 189.360초보다 41.640초 길다.
따라서 스토리보드의 5초/6초 값은 절대 타임라인으로 사용하면 안 된다.
섹션 비중을 계산하는 가중치로만 사용해야 한다.

### 섹션 메타데이터 불일치

`panel_020`부터 `panel_026`까지 파일명은 `chorus`이지만 각 패널 문서 내부
섹션은 `Drop`이다. 자동화에서는 파일명의 섹션 토큰보다 다음 우선순위를
사용한다.

1. 패널 문서 내부의 `섹션`
2. `01_storyboard.md` 테이블의 섹션
3. 파일명의 섹션 토큰

서로 다르면 `metadata_conflict` 경고를 만들고 검토 보고서에 기록한다.

## 5. LRC/SRT 검사

SRT는 총 48개 큐다.

- 실제 가사 큐: 30개
- Suno 섹션/연주 지시문: 18개
- 첫 큐 시작: 29.649초
- 마지막 큐 종료: 186.782초
- 일반 가사 큐 중앙값: 3.630초
- 시간 중첩 쌍: 12개

시간 중첩의 대부분은 같은 시점에 배치된 섹션 태그 및 연주 지시문과 실제
가사다. 화면 자막에는 대괄호 지시문을 제거하고, 섹션 분석용 메타데이터로
별도 보존한다.

### 장시간 큐

| 큐 | 시간 | 길이 | 텍스트 | 판정 |
|---|---|---:|---|---|
| 19 | 70.212-82.150 | 11.938초 | 내 맘을 채워줄 나만의 작은 사치 | 재정렬 권장 |
| 44 | 130.212-174.703 | 44.491초 | 내일 또 만나러 올게 나의 천국 | 자동 사용 금지 |

44번 큐는 다음 타임태그까지 문장을 유지한 다운로드 자막의 구조적 오류일
가능성이 매우 높다. 이 문장을 44.491초 동안 화면에 표시하면 안 된다.
WhisperX 강제 정렬 결과로 실제 발성 종료점을 찾고, 신뢰도가 낮으면
`REVIEW_REQUIRED`로 남긴다.

## 6. 권장 타임라인 처리

1. 0-29.649초는 Intro/Instrumental 이미지 7장에 우선 배정한다.
2. 29.649초 이후에는 정리된 가사 큐와 원문 가사 순서를 기준으로 섹션을
   정렬한다.
3. 130.212-174.703초는 단일 가사 큐로 보지 않고 보컬 정렬 결과와 오디오
   구간 분석으로 나눈다.
4. 174.703초 이후 Outro 이미지를 사용하고 189.360초에서 정확히 종료한다.
5. 각 이미지 지속 시간은 2.5초 이상, 기본 4-8초 범위로 계산한다.
6. 섹션 시간이 이미지 수보다 길면 같은 섹션 안에서 반복하고, 인접 섹션
   이미지를 임의로 가져오지 않는다.

## 7. 프로그램 검증 항목

이 곡은 다음 재사용성 기능을 검증하는 두 번째 실제 사례로 사용한다.

- 한글 곡명과 경로 처리
- 45장 가변 이미지 처리
- 16:9에 가까운 혼합 해상도 처리
- Suno 대괄호 지시문 분리
- 중첩 SRT 큐 정규화
- 비정상 장시간 큐 탐지
- 파일명과 내부 섹션 충돌 탐지
- 29초 이상의 전주 처리
- 음원 끝과 자막 끝의 차이 처리

## 8. 통과 조건

- 45개 이미지가 모두 타임라인에 배치된다.
- 최종 영상 길이가 189.360초와 프레임 오차 이내로 일치한다.
- 대괄호 지시문이 화면 자막으로 출력되지 않는다.
- 44.491초짜리 자막이 정렬 결과에 따라 정상 길이로 축소된다.
- 29번 이미지에서 검은 틈이나 비정상 왜곡이 생기지 않는다.
- 20-26번 패널이 `Drop`으로 정규화된다.
- 자막이 없는 전주와 후주에서도 빈 화면이 발생하지 않는다.

## 9. 결론

`디저트`는 영상 생성에 필요한 자산을 모두 갖춘 두 번째 검증 곡이다.
이미지 수와 음원 길이의 조합도 자동 타임라인에 적합하다. 그러나 원본
SRT와 패널 파일명을 그대로 신뢰해서는 안 된다. 정규화 단계와 WhisperX
재정렬을 거친 뒤에는 재사용형 파이프라인의 유효한 통합 테스트 데이터로
사용할 수 있다.
