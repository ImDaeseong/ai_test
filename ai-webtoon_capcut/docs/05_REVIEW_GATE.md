# 검수와 품질 게이트

## 1. 판정 체계

| 판정 | 의미 |
|---|---|
| PASS | 필수 기능과 출력 품질이 검증되었고 배포 차단 문제가 없음 |
| NEEDS FIX | 핵심 흐름은 동작하지만 수정 후 재검수가 필요함 |
| HOLD | 핵심 기능이 미검증이거나 자막·출력·보안 문제가 배포를 막음 |

현재 판정: `HOLD`

사유: 코드와 실제 실행 결과가 없다.

## 2. 목적 검수

- [ ] 한 문장으로 프로그램 목적을 설명할 수 있다.
- [ ] `ai-webtoon`의 스토리보드를 실제 편집 결과로 연결한다.
- [ ] Remotion을 AI 영상 생성기로 잘못 설명하지 않는다.
- [ ] CapCut과 Remotion의 역할이 중복되지 않는다.
- [ ] 자동화 범위와 사람이 판단할 범위가 분리되어 있다.

## 3. 실행 검수

- [ ] 깨끗한 환경에서 설치 절차를 재현했다.
- [ ] 한 곡을 처음부터 끝까지 처리했다.
- [ ] 음악 길이와 영상 길이가 일치한다.
- [ ] 실패 시 원인을 이해할 수 있는 메시지가 나온다.
- [ ] 원본 파일이 변경되지 않았다.
- [ ] 실행 명령과 실제 결과가 README에 기록되어 있다.

## 4. 자막 검수

- [ ] 첫 보컬 진입이 맞는다.
- [ ] Verse와 Chorus의 대표 cue가 맞는다.
- [ ] Remix/Extend 접합부가 맞는다.
- [ ] 반복 가사를 누락하거나 잘못 복제하지 않았다.
- [ ] 실제로 부르지 않은 가사를 억지로 표시하지 않는다.
- [ ] 긴 음절과 애드리브 처리 정책이 일관적이다.
- [ ] 저신뢰 cue가 명확히 표시된다.
- [ ] 사람이 수정할 수 있는 SRT/CSV가 제공된다.

자동 정렬 결과가 다수 구간에서 0.5~1초 이상 체감상 어긋나거나 잘못된 가사를 다른 구간에 배치하면 `HOLD`로 판정한다. 정량 임계값은 샘플 측정 후 확정한다.

## 5. 영상 출력물 검수

- [ ] 검은 프레임, 깨진 이미지, 누락 이미지가 없다.
- [ ] 이미지 크롭으로 얼굴이나 핵심 피사체가 잘리지 않는다.
- [ ] 12초 이상 정적인 화면이 반복되지 않는다.
- [ ] 빠른 곡에서 컷이 읽을 수 없을 정도로 짧지 않다.
- [ ] 같은 이미지 재사용이 과도하지 않다.
- [ ] 전환이 음악 구조와 크게 충돌하지 않는다.
- [ ] 자막이 화면 밖으로 나가지 않는다.
- [ ] 한글 폰트가 깨지지 않는다.
- [ ] 오디오가 이중 재생되거나 클리핑되지 않는다.
- [ ] 전체 영상 종료가 음악 종료와 자연스럽게 맞는다.

## 6. CapCut 전달 검수

- [ ] 전달 폴더만 보고 가져오기 순서를 알 수 있다.
- [ ] 전체 MP4와 섹션별 MP4의 용도가 구분된다.
- [ ] SRT가 CapCut Desktop에서 정상 import된다.
- [ ] 자막이 개별 텍스트 블록으로 편집 가능하다.
- [ ] 음악 중복 배치 주의사항이 문서에 있다.
- [ ] 절대 경로나 임시 파일이 전달 패키지에 없다.

## 7. 코드 리뷰

- [ ] 입력→처리→출력 흐름을 검토자가 설명할 수 있다.
- [ ] 외부 라이브러리 선택과 라이선스를 확인했다.
- [ ] 함수와 클래스가 단일 책임을 따른다.
- [ ] 하드코딩된 경로와 비밀값이 없다.
- [ ] 빈 `catch`, 무한 재시도, 무제한 모델 호출이 없다.
- [ ] subprocess timeout과 종료 코드 검사가 있다.
- [ ] 설정과 핵심 로직이 분리되어 있다.
- [ ] 테스트가 구현 세부사항이 아니라 계약을 검증한다.
- [ ] Windows 한글·공백 경로가 테스트되었다.
- [ ] 변경된 파일만 범위 있게 수정되었다.

## 8. 보안 검수

- [ ] API 키, 토큰, 비밀번호가 없다.
- [ ] `.env`가 변경 목록에 없다.
- [ ] 개인 음악과 이미지가 공개 저장소에 포함되지 않는다.
- [ ] 로그에 개인 경로와 전체 가사가 불필요하게 노출되지 않는다.
- [ ] 경로 탈출과 임의 명령 실행이 차단된다.
- [ ] 외부 다운로드는 출처, 해시 또는 버전이 관리된다.
- [ ] 공개 가능한 코드와 로컬 전용 데이터가 분리되어 있다.

## 11. 배포 판정

### RELEASE

- 필수 자동 테스트 PASS
- 실제 전체 곡 최소 3종 검수 PASS
- 리믹스 자막 재정렬 샘플 검수 PASS
- CapCut 가져오기 PASS
- P0 없음
- P1이 있으면 사용자 결과를 손상하지 않고 알려진 한계에 기록됨

### CONDITIONAL RELEASE

- 기본 타임라인과 Remotion 렌더는 PASS
- 강제 자막 정렬은 실험 기능으로 꺼져 있음
- 알려진 한계와 수동 검수 절차가 명확함

### HOLD

- 실제 실행 미검증
- 음악과 영상 길이 불일치
- 원본 손상 가능성
- 저신뢰 자막 자동 확정
- CapCut 가져오기 실패
- 보안 또는 민감정보 문제
