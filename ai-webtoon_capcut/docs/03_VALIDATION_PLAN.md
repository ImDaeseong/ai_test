# 검증 계획

## 1. 검증 원칙

- 실행하지 않은 항목은 `UNKNOWN`으로 표시한다.
- 렌더 파일이 생성되었다는 사실만으로 PASS 처리하지 않는다.
- 자동 자막 정확도는 사람이 샘플링 검수해야 한다.
- 검증 결과에는 명령, 날짜, 환경, 입력 fixture, 결과를 기록한다.

현재 모든 항목은 설계 단계이므로 `UNKNOWN`이다.

## 4. 타임라인 검증

### 필수 불변조건

1. 첫 클립 시작은 0이다.
2. 클립은 시간상 역전되지 않는다.
3. 의도된 전환 중첩 외에 빈 구간이나 비정상 중첩이 없다.
4. 모든 duration은 양수다.
5. 마지막 영상 프레임과 음악 마지막 프레임의 차이는 1프레임 이하다.
6. 제외·재사용된 이미지는 보고서에 기록된다.
7. 같은 입력과 설정은 같은 timeline을 생성한다.

| ID | 케이스 | 기대 결과 |
|---|---|---|
| TL-01 | 180초, 이미지 30장 | 평균 약 6초, 음악 길이 일치 |
| TL-02 | 210초, 이미지 42장 | 평균 약 5초, 음악 길이 일치 |
| TL-03 | 180초, 이미지 10장 | 이미지 부족 경고와 정책 적용 |
| TL-04 | 180초, 이미지 100장 | 최소 길이 검사, 일부 제외 후보 |
| TL-05 | 섹션 시간 있음 | 섹션 경계 안에서 패널 배치 |
| TL-06 | 섹션 시간 없음 | 권장 시간 가중치로 전체 배치 |
| TL-07 | 29.97/30fps 변환 | 누적 반올림 오차 허용 범위 이내 |
| TL-08 | 마지막 1ms 오차 | 프레임 규칙으로 결정론적 보정 |

## 5. 자막 검증

### 자동 검증

- cue 시작 < 종료
- cue 정렬 순서
- 음원 범위 밖 cue 없음
- 빈 텍스트 cue 정책 준수
- 지나친 중첩 탐지
- 마지막 cue와 음원 종료 차이 기록
- 기존 파일과 새 파일의 cue 수 변화 기록
- 반복·생략·저신뢰 cue 목록 생성

### 사람 검증 샘플

각 곡에서 최소 다음 구간을 확인한다.

- 첫 보컬 진입
- Verse 한 구간
- 첫 Chorus
- 빠른 발음 또는 랩 구간
- 보컬이 길게 늘어지는 구간
- Remix/Extend 접합부
- 마지막 가사

### 판정

| 상태 | 기준 |
|---|---|
| PASS | 줄 시작과 종료가 자연스럽고 핵심 가사가 실제 가창과 일치 |
| NEEDS FIX | 일부 cue가 빠르거나 늦지만 수정 범위가 제한적 |
| HOLD | 반복·생략을 잘못 해석하거나 다수 cue가 다른 구간에 표시됨 |

숫자 임계값은 기준 곡을 사람이 라벨링한 후 확정한다.

## 6. Remotion 렌더 검증

| ID | 검증 항목 | 기대 결과 |
|---|---|---|

| REN-05 | 모션 프리셋 | 화면 밖 이탈, 검은 여백 없음 |
| REN-06 | 전환 | 깜빡임과 빈 프레임 없음 |
| REN-07 | 재실행 | 기존 결과 보호 또는 명시적 덮어쓰기 |
| REN-08 | 실패 | 불완전 MP4를 성공 결과로 표시하지 않음 |

## 7. CapCut 전달 검증

- MP4가 CapCut Desktop에서 가져오기 가능한가
- SRT가 UTF-8로 정상 표시되는가
- 자막 cue가 개별 편집 가능한가
- 섹션별 MP4 순서가 명확한가
- README에 가져오기 순서가 적혀 있는가
- 원본 음악과 렌더 음악이 중복 재생되지 않도록 안내하는가
- 전달 패키지에 절대 경로나 개인 정보가 포함되지 않는가

## 8. 보안 검증

```powershell
rg -n -i "api[_-]?key|password|secret|token|sk-[a-z0-9]" .
git status --short
git diff --check
```

추가 확인:

- `.env`가 생성·변경되지 않았는가
- 작업 루트 밖 쓰기가 차단되는가
- shell injection 가능한 문자열 결합이 없는가
- 로그에 가사 전체나 사용자 개인 경로가 불필요하게 노출되지 않는가
- 외부 모델 다운로드 위치와 크기가 문서화되었는가
