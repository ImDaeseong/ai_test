# 참고 자료와 설계 적용 근거

## 1. 목적

이 문서는 자막 재정렬 파이프라인이 어떤 연구와 공식 자료를 참고했는지, 각 자료를 어디까지 신뢰할 수 있는지 기록한다.

중요 원칙:

```text
참고 기술이 존재함 != 이 프로젝트의 가창 자막 정확도가 검증됨
```

## 2. WhisperX

### 출처

- 논문: Max Bain, Jaesung Huh, Tengda Han, Andrew Zisserman, “WhisperX: Time-Accurate Speech Transcription of Long-Form Audio”
- 발표: INTERSPEECH 2023
- URL: https://arxiv.org/abs/2303.00747
- DOI: https://doi.org/10.48550/arXiv.2303.00747

### 확인된 내용

WhisperX는 일반 Whisper 장문 전사에서 발생할 수 있는 시간 드리프트, 반복, 환각과 부정확한 발화 타임스탬프 문제를 다룬다. VAD 기반 사전 분할과 강제 음소 정렬을 사용해 단어 단위 타임스탬프를 생성하는 구조다.

### 프로젝트 적용

- 최종 음원에서 보컬 구간 후보를 나누는 VAD 개념
- 초벌 인식 결과에 시간 정보를 부여하는 방법
- 원본 가사와 인식 결과를 연결한 뒤 줄 단위 SRT를 만드는 정렬 단계
- 기존 LRC/SRT를 절대 기준이 아닌 비교 기준으로 사용하는 설계

### 제한

- 논문의 주 대상은 장문 음성 전사이며 노래 가창 전용 정확도 연구가 아니다.
- 멜리스마, 샤우팅, 랩, 코러스 중첩, 강한 음향 효과에서는 성능을 별도 검증해야 한다.
- WhisperX 결과를 그대로 최종 가사로 사용하지 않고 사용자가 보유한 원본 가사를 기준 텍스트로 사용한다.
- 단어 단위 결과가 불안정하면 줄 단위 정렬로 낮춘다.
- 저신뢰 구간은 자동 승인하지 않는다.

## 3. Demucs

### 출처

- 저장소: Meta/Facebook Research Demucs
- URL: https://github.com/facebookresearch/demucs
- 라이선스: 저장소 기준 MIT
- 관련 기술: Hybrid Spectrogram and Waveform Source Separation, Hybrid Transformer Demucs

### 확인된 내용

Demucs는 음악 믹스에서 보컬, 드럼, 베이스, 기타 반주 성분을 분리할 수 있다. `--two-stems=vocals` 방식으로 보컬과 나머지 반주를 나누는 사용 흐름도 제공한다.

### 프로젝트 적용

- 전체 믹스를 직접 인식시키기 전에 보컬 stem을 만드는 선택적 전처리
- 강한 반주가 있는 곡에서 자막 정렬 입력을 개선하기 위한 용도
- 분리 실패 시 원본 믹스를 사용하는 폴백 경로

### 유지보수 위험

- `facebookresearch/demucs` 저장소는 2025년 1월 1일 보관 처리되었고 읽기 전용이다.
- 저장소 설명도 적극적으로 유지보수되지 않는다고 밝히고 있다.
- 따라서 구현 시 해당 저장소를 영구 고정 의존성으로 간주하지 않는다.
- 실제 도입 전 유지되는 포크, 설치 가능 버전, Windows 동작, 모델 라이선스를 다시 검토한다.
- `VocalSeparator` 인터페이스 뒤에 배치해 다른 분리기로 교체 가능하게 한다.

### 품질 제한

- 분리된 보컬에도 반주 누출과 인공적인 음질 손상이 남을 수 있다.
- 보컬 분리가 항상 인식 정확도를 높인다고 가정하지 않는다.
- 원본 믹스와 보컬 stem의 정렬 결과를 비교하는 검증 옵션을 둔다.

## 4. Suno Extend

### 출처

- Suno 공식 도움말: “How do I make my song longer?”
- URL: https://help.suno.com/en/articles/2409601
- 공식 안내 기준 확인일: 2026-06-06

### 확인된 내용

Suno Extend는 사용자가 원곡에서 이어갈 지점을 선택하고 새로운 가사나 스타일 정보를 입력해 뒷부분을 생성하는 방식이다. 선택한 결과는 `Get Whole Song`을 통해 원본 부분과 새 결말을 결합한다.

### 프로젝트 적용

이 동작은 다음 상황을 만들 수 있다.

- 원본과 최종 Whole Song의 길이 변화
- Extend 지점 이후 섹션 길이와 구조 변화
- 새 가사, 반복 가사, 생략 가사
- 접합부 전후 타이밍 변화
- 기존 LRC/SRT와 최종 음원 불일치

따라서 다음 규칙을 적용한다.

1. 최종 Whole Song 파일을 자막과 영상 길이의 기준으로 사용한다.
2. 기존 LRC/SRT가 어느 버전에서 생성되었는지 파일 해시와 함께 기록한다.
3. Extend 또는 Remix 결과는 기본적으로 자막 불일치 가능 상태로 표시한다.
4. Extend 접합부 전후를 사람 검수 필수 구간으로 지정한다.
5. 단순 전체 비율 보정만으로 자막을 확정하지 않는다.

## 5. 결합 설계

```text
Suno 최종 Whole Song
-> 선택적 Demucs 보컬 분리
-> WhisperX 계열 VAD/전사/시간 정렬
-> 원본 가사와 강제 매칭
-> 기존 LRC/SRT와 비교
-> 신뢰도 및 반복·생략 판정
-> 새 SRT와 검수 목록
```

각 도구의 역할:

| 단계 | 도구/자료 | 역할 |
|---|---|---|
| 최종 기준 음원 | Suno Whole Song | 실제 재생 길이와 가창 기준 |
| 보컬 전처리 | Demucs 또는 교체 가능한 분리기 | 반주와 보컬 분리 |
| 시간 추정 | WhisperX 계열 | VAD와 단어/음소 정렬 후보 |
| 텍스트 기준 | 사용자 원본 가사 | 최종 자막 문구 |
| 비교 기준 | 기존 LRC/SRT | 기존 타이밍과 섹션 힌트 |
| 최종 승인 | 사람 검수 | 저신뢰·반복·생략·접합부 확인 |

## 6. 구현 전 검증 과제

- [ ] WhisperX가 한국어 가창 샘플에서 설치·실행되는가
- [ ] 단어 정렬 모델이 대상 언어를 지원하는가
- [ ] 보컬 분리가 원본 믹스보다 실제 정렬 품질을 높이는가
- [ ] 빠른 랩, 발라드, 샤우팅, 코러스 중첩 샘플을 비교했는가
- [ ] Suno Extend 접합부의 반복·생략을 탐지할 수 있는가
- [ ] CPU 폴백의 처리 시간이 실용적인가
- [ ] 모델과 의존성의 라이선스가 배포 방식에 맞는가
- [ ] 실패 시 기존 SRT를 보존하고 사람 검수로 전환하는가

## 7. 인용 원칙

- 문서에는 원문 URL, 제목, 확인일을 남긴다.
- 출처가 말하지 않은 가창 정확도를 임의로 보장하지 않는다.
- 도구 저장소가 보관 또는 비활성 상태이면 유지보수 위험을 함께 기록한다.
- 구현 시 사용한 실제 버전과 모델 이름은 검증 보고서에 별도로 기록한다.
