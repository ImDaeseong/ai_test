# Claude Code Prompt — Windows Network Monitor

## 실행 지시

아래 명세를 바탕으로 **빈 디렉터리에 두 파일을 완성본으로 생성**하라.

1. `network_monitor.py` — 명세를 완전히 구현한 단일 Python 파일
2. `run_monitor.bat` — Windows 배치 파일

**규칙:**
- 설명 없이 파일 생성 작업만 수행한다.
- 코드 외 README, requirements.txt 등 추가 파일은 만들지 않는다.
- 주석은 한국어로 작성하고, 섹션 구분선(`# ──`)을 사용해 가독성을 높인다.
- Python 3.10+ 문법(`str | None`, `from __future__ import annotations`)을 사용한다.
- 외부 라이브러리는 `psutil` 만 사용한다.
- 생성 완료 후 두 파일의 경로를 한 줄씩 출력한다.

---

## 프로젝트 개요

**Windows 전용** 네트워크 모니터링 CLI 도구를 Python으로 작성한다.

- `psutil`로 전체 TCP/UDP 연결 + PID 정보를 **0.5초** 주기로 백그라운드 수집
- **5초** 주기로 터미널에 프로세스별 요약 출력 및 JSON 로그 저장
- TCP LISTEN 포트에 실제 소켓으로 접속해 **HTTP / HTTPS 자동 탐지**
- Windows Raw Socket으로 **HTTP 요청 URL + TLS SNI(HTTPS 도메인)** 실시간 캡처
- 관리자 권한 자동 요청 배치 파일(`run_monitor.bat`) 포함
- Python 3.10+, `psutil` 외 추가 서드파티 패키지 없음

---

## 생성할 파일

```
network_monitor.py   # 메인 Python 소스 (단일 파일)
run_monitor.bat      # 관리자 권한 자동 요청 + psutil 자동 설치 배치
```

런타임에 자동 생성되는 파일:
```
network_monitor.log  # 텍스트 로그 (RotatingFileHandler, 10MB × 3백업)
network_log.json     # JSON 로그 (10MB 초과 시 타임스탬프 suffix로 rename-rotate)
```

---

## 상수 / 설정값

```python
COLLECT_INTERVAL = 0.5      # 백그라운드 수집 주기(초)
DISPLAY_INTERVAL = 5        # 터미널 출력 주기(초)
PROTO_TIMEOUT    = 0.4      # HTTP/HTTPS 탐지 소켓 타임아웃
MAX_WORKERS      = 20       # 웹 프로토콜 탐지 병렬 스레드 수
MAX_LOG_BYTES    = 10 * 1024 * 1024  # 로그 rotate 기준 (10MB)
```

`--collect-interval`, `--display-interval` CLI 인수로 런타임 변경 가능.

---

## 아키텍처 — 스레드 구조

```
main()
 ├─ net-collector (daemon thread)
 │   └─ 0.5초마다 psutil.net_connections() 수집 → _accumulated dict에 누적
 │       └─ 새 TCP LISTEN 포트 발견 시 → web-check ThreadPoolExecutor에 제출
 │
 ├─ http-sniffer (daemon thread)
 │   ├─ sniffer-127.0.0.1  (raw socket, loopback)
 │   └─ sniffer-<외부IP>   (raw socket, 외부 인터페이스)
 │       └─ HTTP 요청 URL + TLS ClientHello SNI 파싱 → _http_log에 append
 │
 └─ main loop (5초 주기)
     ├─ _snapshot_and_group()  → _accumulated 스냅샷 후 PID별 그룹화
     ├─ save_json_log()        → network_log.json 저장
     └─ print_results()        → 터미널 출력
```

---

## 공유 상태

```python
_lock         = threading.Lock()
_accumulated: dict[tuple, dict]   # 수집된 연결 누적 (5초 윈도우, 출력 후 초기화)
_history:     list[dict]          # 세션 전체 연결 이력 (누적, 초기화 없음)
_http_log:    list[dict]          # HTTP/HTTPS 요청 이력 (스니퍼 캡처)
_web_lock     = threading.Lock()
_web_cache:   dict[str, str]      # "ip:port" -> "HTTP"|"HTTPS"
_web_pending: set[str]            # 현재 탐지 진행 중인 포트 키
_stop_event   = threading.Event() # Ctrl+C 종료 시그널
_web_executor = ThreadPoolExecutor(max_workers=MAX_WORKERS, thread_name_prefix="web-check")
```

## 핵심 함수 명세

### `is_admin() -> bool`
`ctypes.windll.shell32.IsUserAnAdmin()` 로 Windows 관리자 권한 확인.

---

### `_is_system_process(name, exe, pid) -> bool`
아래 조건 중 하나라도 해당하면 `True`:
1. `pid in (0, 4)` (System/Idle)
2. `name.lower()` 가 `SYSTEM_PROCESS_NAMES` 집합에 포함
3. `exe` 가 `%WINDIR%` 경로로 시작

`SYSTEM_PROCESS_NAMES`에 포함할 프로세스 이름 목록 (소문자):
```
system, system idle process, idle, registry,
smss.exe, csrss.exe, wininit.exe, services.exe,
lsass.exe, svchost.exe, dwm.exe, winlogon.exe,
fontdrvhost.exe, sihost.exe, taskhostw.exe,
spoolsv.exe, searchindexer.exe, searchhost.exe,
wmiprvse.exe, dllhost.exe, conhost.exe, ctfmon.exe,
ntoskrnl.exe, msdtc.exe, lsm.exe, audiodg.exe,
wlanext.exe, securityhealthservice.exe, runtimebroker.exe,
startmenuexperiencehost.exe, textinputhost.exe, applicationframehost.exe,
msseces.exe, sppsvc.exe, wuauclt.exe, trustedinstaller.exe,
tiworker.exe, msiexec.exe, taskeng.exe, taskhost.exe
```

---

### `check_web_protocol(ip, port, timeout) -> str | None`
실제 소켓으로 HTTP/HTTPS 판별. 순서:
1. `ssl._create_unverified_context()` + `ctx.wrap_socket()` → 성공 시 `"HTTPS"`
2. `HEAD / HTTP/1.1\r\nHost: localhost\r\nConnection: close\r\n\r\n` 전송 후 응답에 `b"HTTP"` 또는 `b"html"` 포함 시 `"HTTP"`
3. 실패 시 `None`

---

### `_connectable_ip(ip) -> str`
- `'0.0.0.0'` 또는 `''` → `'127.0.0.1'`
- `'::'` → `'::1'`
- 그 외 그대로 반환

---

### `_build_proc_cache() -> dict[int, dict]`
`psutil.process_iter(['pid', 'name', 'exe'])` 로 전체 프로세스 순회.  
반환: `{pid: {'name': str, 'exe': str, 'is_system': bool}}`  
`NoSuchProcess`, `AccessDenied` 예외는 조용히 무시.

---

### `_collector(filter_names, include_system) -> None`  *(daemon thread)*
루프 주기: `COLLECT_INTERVAL` 초.

- 프로세스 캐시는 **5회(2.5초)마다** `_build_proc_cache()` 재구성
- `psutil.net_connections(kind="inet")` 호출
- 필터 적용: `include_system=False`이면 시스템 프로세스 skip; `filter_names`가 있으면 해당 이름만 통과
- 연결 키: `(pid, laddr.ip, laddr.port, raddr.ip, raddr.port, proto, status)` 튜플
- **신규 연결**만 `_accumulated`에 추가 + `_history`에도 즉시 append + `logger.info("[NEW] ...")` 기록
- 기존 연결은 `last_seen` 갱신만
- 신규 **TCP LISTEN** 포트 → `_submit_web_check()` 호출 (락 밖에서)

---

### `_submit_web_check(ip, port) -> None`
`_web_cache`와 `_web_pending` 모두에 없을 때만 `_web_executor.submit(_check_and_cache, ip, port)`.

### `_check_and_cache(ip, port) -> None`
`check_web_protocol(_connectable_ip(ip), port)` 호출 후 결과를 `_web_cache[f"{ip}:{port}"]`에 저장.  
`_web_pending`에서 해당 키 제거 (finally).

---

### 패킷 파서 (raw socket 스니퍼용)

#### `_extract_ip_tcp(data) -> tuple | None`
IP+TCP 헤더 파싱 → `(src_ip, src_port, dst_ip, dst_port, payload)`.  
`struct.unpack('!BBHHHBBH4s4s', data[:20])` 로 IP 헤더 파싱, `iph[6] != 6`(비-TCP)이면 None.

#### `_parse_http_request(data) -> dict | None`
payload 첫 4바이트가 HTTP 메서드(`GET `, `POST`, `PUT `, `PATC`, `DELE`, `HEAD`, `OPTI`) 중 하나일 때만 파싱.  
반환 dict 구조:
```python
{
    'timestamp': ...,  # ISO 8601 milliseconds
    'scheme':    'http',
    'src_ip':    ..., 'src_port': ...,
    'dst_ip':    ..., 'dst_port': ...,
    'method':    ..., 'host': ..., 'path': ...,
    'url':       f"http://{host}{path}",
}
```

#### `_parse_tls_sni(data) -> dict | None`
TLS Record `ContentType=0x16`, Handshake `Type=0x01`(ClientHello) 파싱.  
Extension Type `0x0000`(SNI)에서 서버명 추출.  
반환 dict 구조 (HTTP와 동일 필드):
```python
{
    'scheme': 'https', 'method': '(TLS)',
    'path':   '(encrypted)',
    'url':    f"https://{sni}",   # port가 443이면 포트 생략
    ...
}
```

---

### `_sniff_on(bind_ip) -> None`  *(daemon thread)*
```python
sock = socket.socket(AF_INET, SOCK_RAW, IPPROTO_IP)
sock.bind((bind_ip, 0))
sock.setsockopt(IPPROTO_IP, IP_HDRINCL, 1)
sock.ioctl(SIO_RCVALL, RCVALL_ON)
sock.settimeout(1.0)
```
- `recvfrom(65535)` 반복
- `_parse_http_request(raw) or _parse_tls_sni(raw)` → entry 획득 시 `_http_log.append(entry)` + `logger.info`
- `_stop_event` 세트 시 루프 종료, finally에서 `RCVALL_OFF` + `sock.close()`
- 실패(권한 없음 등) 시 `logger.warning` 후 조용히 반환

### `_http_sniffer() -> None`  *(daemon thread)*
- bind_ips 구성: `['127.0.0.1']` + 호스트명으로 `getaddrinfo` 조회한 외부 IPv4들
- 각 IP에 `_sniff_on` daemon thread 시작 후 `join()` 대기

---

### `_snapshot_and_group() -> dict[int, dict]`
`_accumulated` 스냅샷 후 `_accumulated.clear()`.  
PID별 그룹 구조:
```python
{
    pid: {
        'pid': ..., 'name': ..., 'exe': ...,
        'tcp_listen': [{'local_ip', 'local_port', 'web_proto'}, ...],
        'udp_listen': [{'local_ip', 'local_port'}, ...],
        'tcp_estab':  [{'local_ip', 'local_port', 'remote_ip', 'remote_port',
                        'domain', 'first_seen', 'last_seen'}, ...],
    }
}
```
- `tcp_listen`: `web_proto`는 `_web_cache.get(f"{ip}:{port}")`
- `tcp_estab`: `remote_ip`가 공인 IP(127./10./192.168./fe80 아님)이면 `socket.gethostbyaddr()` 역DNS 조회
- 중복 제거: `tcp_listen`, `udp_listen`은 `(local_ip, local_port)` 기준 dict comprehension

---

### `save_json_log(data, scan_count, started_at) -> None`
- 파일이 `MAX_LOG_BYTES` 초과이면 `network_log_YYYYMMDD_HHMMSS.json`으로 rename 후 새 파일 생성
- JSON 구조:
```json
{
  "meta": {
    "started_at": "...", "hostname": "...", "last_updated": "...",
    "total_scans": 0, "display_interval_s": 5, "collect_interval_s": 0.5
  },
  "current":       [...],   // 최근 5초 윈도우 프로세스 요약 (data.values())
  "history":       [...],   // _history (세션 전체 신규 연결 이력)
  "http_requests": [...]    // _http_log (스니퍼 캡처)
}
```

---

### `print_results(data, scan_count) -> None`
터미널 출력 형식:
```
=====================================================================================
  스캔 #N  |  프로세스 N개  |  웹서버 N개  |  수집 0.5s / 표시 5s
=====================================================================================

  PID 1234    node.exe
              C:\Program Files\nodejs\node.exe
              TCP LISTEN : 0.0.0.0:3000 [HTTP],  0.0.0.0:8443 [HTTPS]
              UDP        : 0.0.0.0:53355
              ESTABLISHED: (3건, 이번 주기 관측)
                127.0.0.1:52100 -> api.example.com:443  (최초 2026-04-13T09:00:01)
                ...

[HTTP/HTTPS 요청 — 최근 20건]
  TIME                       SRC                    METHOD   URL
  --------------------------------------------------------------------------------
  2026-04-13T09:00:01.123    127.0.0.1:52100        GET      http://localhost:3000/api
  2026-04-13T09:00:02.456    192.168.1.5:54321      (TLS)    https://api.example.com  ※경로암호화
```
- ESTABLISHED 목록은 `first_seen` 기준 정렬 후 **최대 5건** 출력, 초과분은 `"... 외 N건"`
- `_http_log`의 **최근 20건** 출력

### `_parse_args() -> argparse.Namespace`
| 인수 | 설명 |
|---|---|
| `-p` / `--process NAME` | 모니터링할 프로세스 이름 (여러 번 사용 가능) |
| `--include-system` | Windows 시스템 프로세스도 포함 |
| `--display-interval SEC` | 터미널 출력 주기(기본 5) |
| `--collect-interval SEC` | 백그라운드 수집 주기(기본 0.5) |

`_normalize_filters(names)`: 이름마다 `.exe` suffix 추가 버전도 set에 포함.

---

### `main() -> None`
1. `_parse_args()` 후 전역 `COLLECT_INTERVAL`, `DISPLAY_INTERVAL` 갱신
2. 시작 배너 출력 (관리자 권한, 필터, 로그 경로 등)
3. `is_admin()` False이면 `logger.warning`
4. `net-collector` daemon thread 시작 (`_collector`)
5. `http-sniffer` daemon thread 시작 (`_http_sniffer`)
6. `while True` + `time.sleep(DISPLAY_INTERVAL)` 루프
7. `KeyboardInterrupt` → `_stop_event.set()` + thread join(timeout=2) + 종료 메시지

## run_monitor.bat 요구사항

1. `net session >nul 2>&1` 으로 관리자 권한 확인 → 없으면 PowerShell `Start-Process -Verb RunAs` 로 자신을 재실행 후 `exit /b`
2. `cd /d "%~dp0"` 로 스크립트 디렉터리로 이동
3. `where python` 으로 Python 존재 확인
4. `python -c "import psutil"` 실패 시 `python -m pip install psutil --quiet` 자동 설치
5. `network_monitor.py` 존재 확인
6. 로그 파일 경로 안내 출력 (`network_monitor.log`, `network_log.json`)
7. `python "%~dp0network_monitor.py"` 실행
8. 종료 시 errorlevel 확인 후 메시지 + `pause`

## 제약 및 한계 (코드에 반영 필요)

- `psutil.AccessDenied` 는 조용히 무시 (`pass`)
- Raw socket 스니퍼는 관리자 권한 없으면 `logger.warning` 후 함수 종료 (프로그램 전체는 계속 실행)
- 역방향 DNS 조회는 공인 IP(`127.`, `10.`, `192.168.`, `fe80` 접두사 제외)만 수행
- 프로토콜 탐지 소켓 타임아웃은 `PROTO_TIMEOUT`(0.4초)으로 제한
