# Windows 네트워크 프로토콜 모니터 — 설계 문서

## 목표

현재 PC에서 사용 중인 네트워크 연결을 주기적으로 수집하고,
각 LISTEN 포트에 실제 소켓 통신을 시도하여 **HTTP / HTTPS 여부를 정밀하게 판별**한다.
결과는 터미널 출력 및 JSON 파일로 저장한다.

---

## 핵심 아키텍처

```
main()
 ├─ is_admin()                  관리자 권한 확인
 ├─ while True (5초 주기)
 │   ├─ collect()               네트워크 수집 + 프로토콜 탐지
 │   │   ├─ psutil.net_connections()      모든 TCP/UDP 연결 획득
 │   │   ├─ _build_proc_cache()           PID → {name, exe} 캐시
 │   │   ├─ ThreadPoolExecutor            LISTEN 포트 병렬 프로토콜 스캔
 │   │   │   └─ check_web_protocol()      HTTPS → HTTP 순서로 실제 소켓 탐지
 │   │   └─ 연결별 결과 조립 (domain 역DNS 포함)
 │   ├─ save_json_log()          network_log.json 저장 (10MB rotate)
 │   └─ print_results()          터미널 출력
 └─ KeyboardInterrupt → 종료
```

---

## 수집 항목

| 필드 | 설명 |
|---|---|
| `proto` | TCP / UDP |
| `local_addr` / `local_port` | 로컬 바인딩 주소·포트 |
| `remote_addr` / `remote_port` | 원격 주소·포트 (없으면 null) |
| `status` | LISTEN / ESTABLISHED / TIME_WAIT 등 |
| `pid` | 점유 프로세스 PID |
| `process_name` | 프로세스 이름 |
| `process_exe` | 실행 파일 전체 경로 |
| `is_web` | HTTP 또는 HTTPS 여부 (bool) |
| `protocol` | "HTTP" / "HTTPS" / null |
| `domain` | 원격 IP의 역방향 DNS (공인 IP만) |
| `timestamp` | 수집 시각 (ISO 8601) |

---

## 프로토콜 탐지 방식

1. **HTTPS 탐지**: SSL/TLS handshake 시도 → 성공 시 `"HTTPS"`
2. **HTTP 탐지**: `HEAD / HTTP/1.1` 요청 → 응답에 `HTTP` 또는 `html` 포함 시 `"HTTP"`
3. **ESTABLISHED 포트 추론**: remote_port가 443 → HTTPS, 80 → HTTP로 간주
4. 탐지 실패 시 `null`

모든 LISTEN 포트에 대해 `ThreadPoolExecutor(max_workers=20)`로 병렬 탐지.

## 출력 파일

| 파일 | 내용 |
|---|---|
| `network_log.json` | 마지막 스캔 결과 (meta + connections 배열). 10 MB 초과 시 타임스탬프 suffix로 rotate |
| `network_monitor.log` | 텍스트 로그 (RotatingFileHandler, 10 MB × 3 백업) |

```json

    "hostname": "PC1",

```

## 주요 함수

| 함수 | 역할 |
|---|---|
| `check_web_protocol(ip, port)` | 실제 소켓으로 HTTP/HTTPS 판별 |
| `_build_proc_cache()` | psutil로 PID→{name,exe} 캐시 구성 |
| `collect()` | 연결 수집 + 프로토콜 탐지 통합, 결과 list 반환 |
| `save_json_log(data, ...)` | JSON 파일 저장 + rotate |
| `print_results(data, scan_count)` | 터미널 테이블 출력 |
| `is_admin()` | Windows 관리자 권한 확인 |
| `main()` | 루프 + Ctrl+C 처리 |

## 실행 방법

```bat
run_monitor.bat   # 관리자 권한 자동 요청, psutil 자동 설치
```

또는 직접:

```bash
python network_monitor.py
```

---

## 기술 요구사항

- Python 3.10+ (union type hint `str | None` 사용)
- 필수: `psutil`, `socket`, `ssl`, `json`, `concurrent.futures`, `logging`
- Windows 전용 (`ctypes.windll.shell32.IsUserAnAdmin`)

---

## 제약 및 한계

- `psutil.net_connections()` 는 관리자 권한 없이도 동작하나, 일부 시스템 프로세스의 PID·exe는 `AccessDenied`로 빈 값이 될 수 있음
- 프로토콜 탐지 소켓은 대상 서버에 실제 연결을 시도하므로 방화벽 환경에서 false negative 가능
- 역방향 DNS 조회는 네트워크 상태에 따라 지연 발생 가능 (공인 IP만 조회)
- 브라우저 URL 수집은 포함하지 않음 (Chrome DevTools Protocol은 디버그 포트 활성화 필요)
