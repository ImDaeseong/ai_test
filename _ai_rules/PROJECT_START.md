## 2. 데이터 민감도 분류

> 이 분류가 이후 모든 보안 결정의 기준이 된다

- [ ] **일반** - 공개되어도 무관한 데이터만 처리
- [ ] **내부** - 비공개지만 유출 시 피해 낮음
- [ ] **민감** - 개인정보, 결제정보, 인증정보 포함
- [ ] **기밀** - 영업비밀, 고객 계약, 내부 시스템 접근정보 포함

## 3. 보안 기반 설정 (Day 0 필수)

### 비밀값 관리
- [ ] `.env.example` 생성 (값 없이 키 이름만)
- [ ] `.gitignore`에 `.env`, `*.key`, `*.pem`, `secrets/` 포함 확인
- [ ] 비밀값 저장 방식 결정

```
비밀값 저장 방식:
  개인 프로젝트 → .env 로컬 전용 / 환경변수
  회사 프로젝트 → (예: AWS Secrets Manager / Vault / 환경변수)
```
