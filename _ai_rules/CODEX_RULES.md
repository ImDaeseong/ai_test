# Codex 세션 운용 규칙

> 복사 위치: 프로젝트 루트 `/CODEX_RULES.md`
> 대상: OpenAI Codex CLI
> 참조: VSCODE_AI_RULES.md (Claude Code용), HANDOFF_TEMPLATE.md (도구 전환용)

---

## Claude Code와 핵심 차이

| 항목 | Claude Code | Codex |
|---|---|---|
| CLAUDE.md 자동 로드 | ✅ 자동 | ❌ 수동 지정 필요 |
| 파일 참조 방식 | `@파일명` | 직접 파일명 언급 |
| 컨텍스트 초기화 | `/clear` | 새 세션 시작 |
| 대화 이어가기 | `--continue` | 없음 (세션 내 연속만) |
| 터미널 명령 실행 | 통합 | 기본 강점 |

**Codex는 CLAUDE.md를 모른다.** 매 세션 시작 시 컨텍스트를 직접 제공하지 않으면 프로젝트 규칙 없이 작업하게 된다.

## 세션 시작 — 신규 세션 부팅 프롬프트

```
다음 파일들을 순서대로 읽어줘:
1. START_HERE.md 또는 프로젝트 시작 문서  (있는 경우)
2. AGENTS.md 또는 CLAUDE.md
3. SECURITY_RULES.md
4. HANDOFF.md 또는 SESSION_HANDOFF.md  (있는 경우)
5. [작업할 프로젝트]/CLAUDE.md  (해당하는 경우)

읽은 후 아래 작업을 시작해줘:
목표: [한 문장]
완료 기준: [이렇게 되면 끝]
수정 허용 파일: [파일 목록]
```

### 세션 시작 체크리스트

- [ ] 현재 git 상태 확인 (`git status`, `git log --oneline -3`)
- [ ] 세션 목적 한 문장 정의
- [ ] 완료 기준 정의
- [ ] 수정 허용 파일 목록 제한

## 세션 시작 — HANDOFF 이어받기

HANDOFF.md 또는 SESSION_HANDOFF.md가 있는 경우:

```
다음 파일들을 순서대로 읽어줘:
1. HANDOFF.md
2. CLAUDE.md
3. [HANDOFF.md의 "먼저 읽어야 할 파일" 목록]

읽은 후 HANDOFF.md의 "다음 할 일" 첫 번째 항목부터 이어서 시작해줘.
작업 범위는 HANDOFF.md의 "현재 목표"로 제한해줘.
```

## 보안 규칙 (CLAUDE.md 자동 로드 없으므로 세션마다 적용)

- **API Key 절대 금지**: 어떤 키·토큰도 코드에 직접 쓰지 않는다
- **.env 파일 보호**: `.env`를 생성·수정하지 않는다. `.env.example`만 수정한다
- **파괴적 작업 금지**: 사용자 승인 없이 중요 데이터 파일을 삭제·덮어쓰지 않는다
- **범위 준수**: 지정한 파일 목록 외 파일은 수정하지 않는다

## 컨텍스트 관리

### Codex는 `/clear`가 없다

새 태스크로 전환할 때 → **새 세션 시작** + 부팅 프롬프트 재사용

### 컨텍스트 오염 신호 — 하나라도 해당하면 새 세션

| 신호 | 행동 |
|---|---|
| 같은 에러 3회 이상 반복 | 세션 종료 → HANDOFF 작성 → 새 세션 |
| 이전 결정을 무시하기 시작 | 세션 종료 → HANDOFF 작성 → 새 세션 |
| 답변이 CLAUDE.md 규칙과 어긋남 | CLAUDE.md 다시 읽으라고 요청 |
| 수정 파일이 지정 범위 벗어남 | 즉시 중단, `git restore [파일명]` |

### 같은 방법으로 3회 실패 시

```
지금까지 시도한 방법:
1. [시도 1]
2. [시도 2]
3. [시도 3]

원인 분석만 해줘. 코드 수정하지 마.
```
