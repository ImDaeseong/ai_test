# 프로젝트: 서울 가산동 날씨 알리미 (Discord & Telegram 통합 버전)

[수행 지침]
기상청 초단기실황 API를 사용하여 가산동(nx=58, ny=125) 날씨를 가져오고, 이를 디스코드와 텔레그램 두 채널로 동시에 서비스할 수 있는 구조를 구축하라.

1. 환경 및 규칙 설정:
   - .claude/rules/api.md: 기상청 API 명세 및 10분 지연 호출 로직 기록.
   - .env.example: WEATHER_SERVICE_KEY, DISCORD_TOKEN, TELEGRAM_TOKEN, TELEGRAM_CHAT_ID 항목 생성.
   - requirements.txt: aiohttp, python-dotenv, discord.py, python-telegram-bot 추가.

2. 공통 모듈 및 API 클라이언트:
   - weather_client.py: 기상청 데이터를 가져와서 두 플랫폼 모두에서 쓸 수 있는 공통 텍스트/데이터 포맷으로 가공하는 비동기 클래스.

3. 플랫폼별 구현:
   - discord_bot.py: 슬래시 커멘드(/가산날씨)를 통해 Embed 형식으로 응답하는 봇 구현.
   - telegram_bot.py: 텔레그램 메시지 루프를 통해 '/weather' 명령에 응답하거나 알림을 보내는 봇 구현.
   - main.py: asyncio.gather를 사용하여 디스코드 봇과 텔레그램 봇을 동시에 실행하는 엔트리 포인트 작성.

4. 품질 관리 (QA):
   - 한쪽 플랫폼의 토큰이 없더라도 다른 쪽은 정상 작동하도록 예외 처리.
   - 로깅 시스템(loguru 등)을 추가하여 각 플랫폼의 동작 상태를 추적 가능하게 함.

작업 완료 후, 각 플랫폼의 봇 토큰 발급 방법과 .env 설정법을 안내하며 마무리하라. 구현을 시작하라.
