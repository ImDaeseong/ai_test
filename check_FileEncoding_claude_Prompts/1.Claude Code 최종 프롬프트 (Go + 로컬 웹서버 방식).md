목표:
Windows에서 실행 가능한 단일 EXE 형태의 "파일 인코딩 검사 도구"를 Go로 만들어라.
이 프로그램은 로컬 웹서버를 실행하고, 브라우저 UI를 통해 폴더를 입력받아
해당 폴더의 소스 파일 인코딩을 분석하여 결과를 표시한다.

전체 구조:
- Go 단일 실행 파일 (main.go)
- 내장 웹 서버 (net/http)
- HTML UI는 Go 코드 내에 embed 하거나 /static 폴더로 제공
- 실행 시 자동으로 브라우저 오픈 (http://localhost:8080)

-----------------------------------
[핵심 기능]
-----------------------------------

1. 폴더 스캔
- 사용자가 웹페이지에서 폴더 경로 입력
- 서버는 해당 경로를 기준으로 재귀적으로 파일 탐색
- 대상 확장자:
  .cpp, .h, .hpp, .c, .rc

2. 인코딩 판별 (정확히 구현)
다음 순서로 판단:

(1) BOM 검사
- UTF-8 BOM → "UTF-8 BOM"
- UTF-16 LE → "UTF-16 LE"
- UTF-16 BE → "UTF-16 BE"

(2) BOM 없을 경우
- UTF-8 유효성 검사
  → 성공: "UTF-8 (No BOM)"
  → 실패: "EUC-KR (CP949)"

※ 절대 인코딩 변환하지 말 것. 반드시 raw byte 기반 분석만 수행.

3. 결과 출력 (웹 UI)
테이블 형태:

- 파일명
- 전체 경로
- 인코딩 타입
- BOM 여부 (Yes/No)

4. UI 기능
- 입력창: 폴더 경로
- 버튼: [스캔]
- 로딩 표시 (간단한 spinner 또는 텍스트)
- 필터 기능:
  - 전체
  - UTF-8만
  - EUC-KR만

5. 성능
- 파일 스캔은 goroutine 사용 (병렬 처리)
- 결과는 JSON으로 반환

-----------------------------------
[웹 API 설계]
-----------------------------------

GET /
→ index.html 반환

POST /scan
Request:
{
  "path": "C:\\project"
}

Response:
[
  {
    "name": "main.cpp",
    "path": "C:\\project\\main.cpp",
    "encoding": "EUC-KR (CP949)",
    "bom": false
  }
]

-----------------------------------
