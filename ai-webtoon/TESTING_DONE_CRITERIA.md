# Testing Done Criteria

## Given-When-Then

1. Given 장르와 BPM이 다른 곡들, When 공연 프로필을 선택하면, Then 최소 5개 서로 다른 공연 문법이 선택된다.
2. Given 같은 곡과 패널 번호, When 변형을 반복 선택하면, Then 결과가 항상 동일하다.
3. Given 여러 패널, When 카메라와 조명 변형을 선택하면, Then 하나의 고정 문구만 반복되지 않는다.
4. Given 생성된 패널 프롬프트, When 출처 아티스트명을 검사하면, Then 실존 밴드명이 포함되지 않는다.

## Automated Checks

- 기존 단위 테스트 전체
- 신규 프로필 선택·결정론·데이터 참조 무결성 테스트
- Python 구문 검사
- 대표 곡 생성 및 `validate`

## Human Q3/Q4

- Q3: `UPGRADE`, 발라드 1곡, 전자음악 1곡의 스타일 문서를 나란히 비교한다.
- Q3: 카메라·조명·무대 배치가 곡별로 구별되는지 확인한다.
- Q4: 출처 아티스트 복제, 로고, 실존 얼굴, 고유 의상 재현이 없는지 확인한다.

## Done

- P0/P1 오류 0개
- 자동 테스트 통과
- 대표 출력 검증 통과
- 사람 검수 전에는 "배포 완료"로 표시하지 않는다.
