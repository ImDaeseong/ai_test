Implement modules/web_scanner.py for DefenseScan with these checks:

1. Security Headers (Missing/Misconfigured)
   - Content-Security-Policy (detect unsafe-inline, unsafe-eval)
   - Strict-Transport-Security
   - X-Frame-Options
   - X-Content-Type-Options
   - Referrer-Policy
   - Permissions-Policy
   - Information leak headers (Server, X-Powered-By)

2. SSL/TLS Analysis
   - Certificate expiry (warn if < 30 days, critical if expired)
   - Certificate verification failure
   - Weak protocol detection (TLS 1.0, 1.1, SSLv2, SSLv3)
   - HTTP vs HTTPS detection

3. Directory Listing Detection
   - Check common paths: /admin/, /uploads/, /backup/, /logs/, /config/
   - Detect listing indicators in response body

4. HTTP Response Analysis
   - Status codes anomaly
   - Response time measurement
   - Redirect chain analysis

Design:
- Class-based structure (one class per check)
- ThreadPoolExecutor for concurrent scanning
- Each finding must include: check, risk level, title, detail, recommendation, evidence
- Risk levels: Critical / High / Medium / Low / Info
