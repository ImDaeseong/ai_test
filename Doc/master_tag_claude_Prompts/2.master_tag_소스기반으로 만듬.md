# 🎛️ Claude 전용 프롬프트 — Suno AI 전문 마스터링 엔지니어링 시스템

---

## [역할 지정]

너는 **멀티 스택(C#, Go, Python) 시니어 개발자**이자 **전문 사운드 엔지니어**다.  
Suno AI 음원의 고질적 결함을 **주파수 데이터(FFT) 및 알고리즘** 관점에서 분석하고,  
수익 파이프라인 자동화를 위한 **로컬 완결형 마스터링 시스템**을 설계·구현하라.  
설명은 코드 내 **한글 주석**으로만 대체하며, 불필요한 서술은 생략한다.

---

## 🚨 절대 원칙 (반드시 준수)

| 원칙 | 내용 |
|------|------|
| **No External API** | 모든 오디오 처리는 로컬 라이브러리(`Pedalboard`, `Librosa`, `Pyloudnorm`)만 사용. 유료 외부 API 사용 금지. |
| **Data-Driven DSP** | 감성 접근 금지. FFT·레이어링 수치 기반으로 DSP 파라미터를 산출하고 근거를 주석에 명시. |
| **계층 분리 아키텍처** | UI(`index.html`) / Server(`server.py`) / Engine(`main.py`)을 명확히 분리하여 확장성을 확보. |
| **실시간 모니터링** | 처리 진행 상태는 **SSE(Server-Sent Events)** 스트림으로 UI에 단계별 JSON 전달. |
| **즉시 배포 가능** | `PyInstaller(.spec)` 설정을 포함하여 단일 실행 파일로 배포 가능한 구조 유지. |
| **파라미터 분리** | 모든 DSP 계수는 하드코딩 금지. 함수 인자로 분리하고 기본값과 조정 범위를 주석에 명시. |

## 🛠️ 생성 대상 파일 (5종 — 전부 완성 코드로 출력)

### 1. `main.py` — Core Audio Engine

다음 파이프라인을 순서대로 구현한다.

**① 오디오 분석 (librosa)**
- `librosa.load()`로 스테레오 유지, 원본 샘플레이트 사용
- Peak(dBFS), RMS(dBFS), 재생 시간(초), 샘플레이트 정밀 측정
- 측정값을 `dict`로 반환하여 상위 파이프라인에서 활용 가능하게 설계

**② Gain Staging**
- 입력 Peak를 `-6.0 dBFS`로 선형 조정하여 DSP 헤드룸 확보
- 클리핑 방지를 위해 DSP 체인 진입 전 반드시 적용

**③ DSP 체인 (Pedalboard) — Suno AI 최적화**

```
HighpassFilter     → 30Hz  미만 서브 베이스 노이즈 / DC 오프셋 제거
LowShelfFilter     → 100Hz 저역 보강 (+3dB) — 무게감·펀치감 복원
PeakFilter (Cut)   → 350Hz 중저역 혼탁도(Muddiness) 제거 (-3dB)
PeakFilter (Boost) → 3kHz  보컬·스네어 존재감(Presence) 강화 (+1.5dB)
HighShelfFilter    → 10kHz 에어(Air) 대역 복원 (+4dB) — Suno AI 최대 약점 보완
Compressor         → 다이나믹 밀도 강화 (Threshold: -18dB, Ratio: 3:1)
Limiter            → -1.0 dBTP 피크 방지 (스트리밍 표준)
```

- 각 이펙트 파라미터에 **조정 가능 범위 및 Suno AI 보정 근거** 한글 주석 필수

**④ LUFS 정규화 (pyloudnorm)**
- ITU-R BS.1770-4 기반 통합 라우드니스 측정
- `-14.0 LUFS` (YouTube / Spotify 표준)로 강제 정규화
- 무음·측정 불가 파일은 경고 후 건너뜀 처리

**⑤ 저장 및 콜백**
- `Pedalboard AudioFile`로 WAV 고음질 저장
- `on_progress: Callable[[dict], None]` 콜백으로 Web UI / Go 스케줄러 연동
- 이벤트 타입: `step` / `analysis` / `lufs` / `done` / `error`

**CLI 진입점**
- `sys.argv[1]` 또는 `input()`으로 파일 경로 수신
- 라이브러리 미설치 시 `pip install` 안내 후 `sys.exit(1)`

### 2. `server.py` — Flask Web Server

- `POST /master` : 파일 업로드 → UUID 작업 생성 → 백그라운드 스레드 실행 → `job_id` 반환
- `GET /progress/<job_id>` : SSE 스트림으로 실시간 이벤트 JSON 전달 (heartbeat 포함)
- `GET /download/<job_id>` : 처리 완료 WAV 파일 다운로드
- `GET /` : `index.html` 서빙
- 업로드/출력 디렉토리는 `uploads/`, `outputs/` 자동 생성
- 포트: `5000`, `threaded=True`, `debug=False`

---

### 3. `index.html` — Modern Dark UI (단일 파일, 외부 CDN 의존 없음)

- **디자인**: 다크 모드, CSS 변수 기반 컬러 시스템, 반응형 레이아웃
- **드래그 앤 드롭 존**: `hover` / `has-file` 상태 시각 전환, 파일명·용량 표시
- **실시간 Step Indicator**: 4단계 진행 표시 (대기 → active 맥박 애니메이션 → done ✓)
- **분석 대시보드**: Peak, RMS, 재생 시간, 샘플레이트, LUFS 변화량(Before → After) 카드 출력
- **다운로드 버튼**: 처리 완료 시 자동 활성화
- **오류 배너 + 리셋 버튼**: 모든 상태 초기화

### 5. `SunoMastering.spec` — PyInstaller 빌드 설정

- `onefile=True`, `console=False` (GUI 모드)
- `server.py`를 진입점으로 하여 Flask + 모든 의존성 포함
- `index.html`을 `datas`에 포함 (`--add-data`)
- `librosa`, `pedalboard` 등 숨겨진 임포트(`hiddenimports`) 명시
- Windows 10/11 실행 환경 기준
