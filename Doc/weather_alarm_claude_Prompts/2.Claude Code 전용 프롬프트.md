# Claude Code 전용 프롬프트 — 공공 API 기반 멀티 플랫폼 알림 봇

> **용도**: 공공/외부 API 데이터를 수집하여 Discord·Telegram 등 복수 채널로 동시에 전송하는  
> Python 비동기 봇 프로젝트를 새로 시작할 때 이 프롬프트를 Claude Code에 그대로 붙여넣어 사용한다.  
> 프로젝트별 고유값(API URL, 좌표, 봇 명령어 이름 등)만 수정하면 된다.

---

## [역할 지시]

너는 숙련된 Python 백엔드 엔지니어다.  
아래 명세를 읽고 **처음부터 완전한 프로젝트**를 구현하라.  
구현 전에 파일 목록과 각 파일의 역할을 먼저 출력하고, 승인 없이 바로 코딩을 시작하라.

---

## [프로젝트 개요]

`{외부 API 이름}`에서 `{데이터 종류}` 데이터를 주기적으로 수집하여  
**Discord**와 **Telegram** 두 채널로 동시에 서비스하는 Python 비동기 봇 애플리케이션.

> 예시(날씨 알리미): 기상청 초단기실황 API로 서울 가산동 날씨를 조회하여  
> Discord 슬래시 커맨드와 Telegram `/weather` 명령으로 제공.

## [구현 명세]

### 1. `.claude/rules/api.md` — API 명세 문서

프로젝트가 호출하는 외부 API의 **엔드포인트, 요청 파라미터, 응답 구조, 코드표**를 기록한다.  
Claude Code는 이 파일을 자동으로 참조하므로 API 관련 질문 시 별도 설명 없이 정확하게 동작한다.

포함 항목:
- GET/POST 엔드포인트 URL
- 필수·선택 파라미터 표 (이름, 설명, 예시값)
- 응답 JSON 구조 (`response.header.resultCode`, `response.body.items.item[]` 등)
- 상태·분류 코드표 (코드 → 한국어 설명)
- 데이터 지연/제한 등 호출 시 주의사항

### 2. `{domain}_client.py` — API 클라이언트

**반드시 아래 패턴을 따른다.**

```python
import aiohttp
from dataclasses import dataclass
from typing import Optional
from loguru import logger

API_URL = "{외부 API 엔드포인트}"

# 코드 → 한국어 매핑 딕셔너리
CODE_MAP = { ... }


@dataclass
class {Domain}Data:
    """API 응답을 담는 불변 데이터 모델."""
    field1: str
    field2: str
    # ...

    def format_text(self) -> str:
        """Discord/Telegram 양쪽에서 공통으로 쓰는 텍스트 포맷."""
        return (
            f"헤더 라인\n"
            f"필드1: {self.field1}\n"
            f"필드2: {self.field2}\n"
        )


class {Domain}Client:
    def __init__(self, service_key: str):
        self.service_key = service_key

    @staticmethod
    def _build_params(service_key: str) -> dict:
        """API 요청 파라미터를 생성한다."""
        # 시각 지연 등 특수 로직이 있으면 여기서 처리
        return { ... }

    async def fetch(self) -> Optional[{Domain}Data]:
        params = self._build_params(self.service_key)
        logger.info(f"API 요청: {params}")
        try:
            timeout = aiohttp.ClientTimeout(total=10)
            async with aiohttp.ClientSession(timeout=timeout) as session:
                async with session.get(API_URL, params=params) as resp:
                    resp.raise_for_status()
                    data = await resp.json(content_type=None)
            logger.debug(f"API 응답 원본: {data}")

            # resultCode 체크
            result_code = data.get("response", {}).get("header", {}).get("resultCode")
            if result_code != "00":
                result_msg = data.get("response", {}).get("header", {}).get("resultMsg", "알 수 없는 오류")
                logger.error(f"API 오류 응답: resultCode={result_code}, resultMsg={result_msg}")
                return None

            # 파싱
            items = data["response"]["body"]["items"]["item"]
            obs = {item["category"]: item["obsrValue"] for item in items}
            logger.info(f"API 파싱 성공: 항목 수={len(obs)}")

            return {Domain}Data(
                field1=obs.get("FIELD1", "N/A"),
                field2=obs.get("FIELD2", "N/A"),
            )
        except aiohttp.ClientError as e:
            logger.error(f"네트워크 오류: {e}")
        except (KeyError, ValueError) as e:
            logger.error(f"응답 파싱 오류: {e}")
        except Exception as e:
            logger.error(f"데이터 조회 실패: {e}")
        return None
```

### 3. `discord_bot.py` — Discord 봇

**반드시 아래 패턴을 따른다.**

```python
import discord
from discord import app_commands
from loguru import logger
from {domain}_client import {Domain}Client, {Domain}Data


class Discord{Domain}Bot(discord.Client):
    def __init__(self, client: {Domain}Client, channel_id: int = 0):
        intents = discord.Intents.default()
        super().__init__(intents=intents)
        self.tree = app_commands.CommandTree(self)
        self.client = client
        self.channel_id = channel_id
        self._register_commands()

    def _register_commands(self):
        @self.tree.command(name="{슬래시_커맨드_이름}", description="{설명}")
        async def _cmd(interaction: discord.Interaction):
            await interaction.response.defer()
            data: {Domain}Data | None = await self.client.fetch()
            if data is None:
                await interaction.followup.send("❌ 데이터를 가져오지 못했습니다. 잠시 후 다시 시도해주세요.")
                return
            embed = discord.Embed(title="제목", description="설명", color=discord.Color.blue())
            # embed.add_field(name="항목", value=data.field1, inline=True)
            embed.set_footer(text="데이터 출처: {API 이름}")
            await interaction.followup.send(embed=embed)
            logger.info(f"Discord 전달 완료 | 요청자={interaction.user}")

    async def setup_hook(self):
        await self.tree.sync()
        logger.info("Discord 슬래시 커맨드 전역 동기화 완료")

    async def on_ready(self):
        logger.info(f"Discord 봇 연결됨: {self.user} (ID: {self.user.id})")
        if not self.channel_id:
            return
        try:
            channel = await self.fetch_channel(self.channel_id)
        except Exception as e:
            logger.warning(f"Discord 채널 조회 실패: {e}")
            return
        data = await self.client.fetch()
        if data is None:
            return
        embed = discord.Embed(title="시작 알림 제목", color=discord.Color.blue())
        await channel.send(embed=embed)
        logger.info(f"Discord 시작 알림 전송 완료: channel_id={self.channel_id}")
```

### 4. `telegram_bot.py` — Telegram 봇

**반드시 아래 패턴을 따른다.**

```python
import asyncio
from telegram import Update
from telegram.ext import Application, CommandHandler, ContextTypes
from loguru import logger
from {domain}_client import {Domain}Client


class Telegram{Domain}Bot:
    def __init__(self, token: str, chat_id: str, client: {Domain}Client):
        self.token = token
        self.chat_id = chat_id
        self.client = client
        self.app = Application.builder().token(token).build()
        self.app.add_handler(CommandHandler("data", self._data_command))
        self.app.add_handler(CommandHandler("start", self._start_command))

    async def _start_command(self, update: Update, _context: ContextTypes.DEFAULT_TYPE):
        await update.message.reply_text("안녕하세요! {서비스명} 알리미입니다.\n\n/data — 현재 데이터 조회")

    async def _data_command(self, update: Update, _context: ContextTypes.DEFAULT_TYPE):
        data = await self.client.fetch()
        if data is None:
            await update.message.reply_text("❌ 데이터를 가져오지 못했습니다. 잠시 후 다시 시도해주세요.")
            return
        await update.message.reply_text(data.format_text())
        logger.info(f"Telegram 전달 완료 | 요청자={update.effective_user.username}")

    async def send_notification(self, message: str):
        if not self.chat_id:
            logger.warning("TELEGRAM_CHAT_ID가 없어 알림을 보낼 수 없습니다.")
            return
        try:
            await self.app.bot.send_message(chat_id=self.chat_id, text=message)
            logger.info(f"Telegram 알림 전송 완료: chat_id={self.chat_id}")
        except Exception as e:
            logger.error(f"Telegram 알림 전송 실패: {e}")

    async def run(self):
        async with self.app:
            await self.app.start()
            logger.info("Telegram 봇 폴링 시작")
            await self.app.updater.start_polling(drop_pending_updates=True)
            data = await self.client.fetch()
            if data:
                await self.send_notification(data.format_text())
            else:
                logger.warning("Telegram 시작 알림: 데이터 조회 실패")
            try:
                await asyncio.Event().wait()
            finally:
                await self.app.updater.stop()
                await self.app.stop()
```

### 5. `main.py` — 진입점

**반드시 아래 패턴을 따른다.**

```python
import asyncio
import io
import os
import sys

import discord
from dotenv import load_dotenv
from loguru import logger

load_dotenv()

# Windows cp949 환경에서 한글·이모지 깨짐 방지
if hasattr(sys.stdout, "buffer"):
    sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding="utf-8", errors="replace")

os.makedirs("logs", exist_ok=True)
logger.remove()
logger.add(
    sys.stdout,
    format="<green>{time:YYYY-MM-DD HH:mm:ss}</green> | <level>{level: <8}</level> | <cyan>{name}</cyan> - <level>{message}</level>",
    level="INFO",
)
logger.add(
    "logs/{프로젝트명}.log",
    mode="w",
    level="DEBUG",
    encoding="utf-8",
)

from {domain}_client import {Domain}Client
from discord_bot import Discord{Domain}Bot
from telegram_bot import Telegram{Domain}Bot

SERVICE_KEY    = os.getenv("{DOMAIN}_SERVICE_KEY", "")
DISCORD_TOKEN  = os.getenv("DISCORD_TOKEN", "")
DISCORD_CHANNEL_ID = int(os.getenv("DISCORD_CHANNEL_ID", "0"))
TELEGRAM_TOKEN = os.getenv("TELEGRAM_TOKEN", "")
TELEGRAM_CHAT_ID   = os.getenv("TELEGRAM_CHAT_ID", "")


async def run_discord(client: {Domain}Client):
    bot = Discord{Domain}Bot(client, DISCORD_CHANNEL_ID)
    try:
        logger.info("Discord 봇 시작 중...")
        await bot.start(DISCORD_TOKEN)
    except discord.LoginFailure:
        logger.error("Discord 토큰이 유효하지 않습니다.")
    except Exception as e:
        logger.error(f"Discord 봇 오류: {e}")


async def run_telegram(client: {Domain}Client):
    bot = Telegram{Domain}Bot(TELEGRAM_TOKEN, TELEGRAM_CHAT_ID, client)
    try:
        logger.info("Telegram 봇 시작 중...")
        await bot.run()
    except Exception as e:
        logger.error(f"Telegram 봇 오류: {e}")


async def main():
    if not SERVICE_KEY:
        logger.error("{DOMAIN}_SERVICE_KEY가 설정되지 않았습니다. .env 파일을 확인하세요.")
        sys.exit(1)

    client = {Domain}Client(SERVICE_KEY)

    # 시작 시 데이터 조회 확인
    data = await client.fetch()
    if data:
        for line in data.format_text().splitlines():
            logger.info(line)
    else:
        logger.warning("초기 데이터 조회 실패 — API 키 또는 네트워크를 확인하세요.")

    tasks = []
    if DISCORD_TOKEN:
        tasks.append(run_discord(client))
    else:
        logger.warning("DISCORD_TOKEN이 없습니다. Discord 봇을 건너뜁니다.")

    if TELEGRAM_TOKEN:
        tasks.append(run_telegram(client))
    else:
        logger.warning("TELEGRAM_TOKEN이 없습니다. Telegram 봇을 건너뜁니다.")

    if not tasks:
        logger.error("실행할 봇이 없습니다. 토큰을 .env에 설정하세요.")
        sys.exit(1)

    logger.info(f"{len(tasks)}개 봇 동시 실행 시작 (Ctrl+C로 종료)")
    await asyncio.gather(*tasks, return_exceptions=True)


if __name__ == "__main__":
    asyncio.run(main())
```

## [품질 요구사항 — 체크리스트]

구현 완료 후 아래 항목을 모두 충족하는지 검토하라.

- [ ] `{DOMAIN}_SERVICE_KEY` 없으면 `sys.exit(1)` 즉시 종료
- [ ] Discord 토큰 / Telegram 토큰 중 하나만 있어도 해당 봇만 단독 실행
- [ ] 모든 예외는 loguru로 캐치 — 프로세스 전체가 죽지 않도록 처리
- [ ] Windows cp949 환경에서 이모지·한글 깨짐 방지 (UTF-8 stdout 리래핑)
- [ ] 로그: stdout(INFO 이상) + `logs/*.log`(DEBUG 이상, UTF-8, mode="w") 이중 출력
- [ ] `aiohttp.ClientTimeout(total=10)` 설정으로 무한 대기 방지
- [ ] API resultCode `"00"` 체크 후 오류 코드는 error 로그 출력 후 `None` 반환
- [ ] `.env`는 `.gitignore`에 포함, `.env.example`만 커밋
- [ ] `logs/` 디렉터리는 `.gitignore`에 포함

## [토큰 발급 가이드 — 출력 템플릿]

구현 완료 후 아래 가이드를 사용자에게 출력하라:

1. **{외부 API} 키**: `{발급 URL}` → API 신청 → serviceKey 발급
2. **Discord 봇 토큰**:
   - `https://discord.com/developers/applications` → New Application → Bot → Token 복사
   - OAuth2 > URL Generator: `bot` + `applications.commands` 스코프 → 서버 초대
   - DISCORD_CHANNEL_ID: 채널 우클릭 > 채널 ID 복사 (개발자 모드 필요)
3. **Telegram 봇 토큰**:
   - @BotFather → `/newbot` → 토큰 발급
   - TELEGRAM_CHAT_ID: @userinfobot에 메시지 전송 후 ID 확인
4. `.env` 파일에 값 입력 후 `python main.py` 실행

## [실제 적용 예시 — 날씨 알리미]

이 템플릿을 기상청 초단기실황 API 기반 프로젝트에 적용한 사례:

| 템플릿 변수 | 실제 값 |
|---|---|
| `{domain}` | `weather` |
| `{Domain}` | `Weather` |
| `{DOMAIN}` | `WEATHER` |
| `{외부 API 이름}` | 기상청 초단기실황 API |
| `{슬래시_커맨드_이름}` | `가산날씨` |
| API URL | `http://apis.data.go.kr/1360000/VilageFcstInfoService_2.0/getUltraSrtNcst` |
| 특수 로직 | 10분 지연 — `now - 10min` 후 분 단위 내림 |

완성된 소스는 `weather_alarm` 프로젝트 참조.
