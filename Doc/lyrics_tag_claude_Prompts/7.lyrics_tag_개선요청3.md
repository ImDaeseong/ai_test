현재 start 계산 로직을 점검하고 수정하라.

---

# 문제

end < start 문제는 해결되었지만
타임스탬프 정확도가 여전히 맞지 않다.

---

# 확인할 것

start 계산 부분에서:

* previous_start 사용 여부
* 누적 보정 여부

---

# 수정 지시

1. previous_start 기반 보정 제거

2. 누적 보정 제거

3. start 계산은 다음으로 통일:

adjusted_start = max(0, segment_start - 0.3)

---

# 목표

* 모든 라인의 start는 독립적으로 계산
* 시간 드리프트 완전 제거

---

수정 후 다시 테스트하라.
