# Node.js FFmpeg Lyric Video Generator 최종 프로젝트 생성 프롬프트

아래 요구사항을 그대로 구현하라. 목표는 로컬 폴더의 가사, 오디오, 배경 이미지를 입력받아 실제로 재생 가능한 MP4 리릭 비디오를 생성하는 Node.js 프로젝트를 만드는 것이다. 최종 결과는 Windows 환경에서 `run.bat` 또는 `npm run lyric-video -- --clean` 한 번으로 생성되고, `npm run validate:media`로 검증되어야 한다.

## 핵심 목표

로컬 Node.js 기반 리릭 비디오 생성 파이프라인을 구현한다.

입력:

```text
input/
  lyrics.srt 또는 lyrics.lrc 또는 lyrics.json
  audio.wav 또는 audio.mp3
  background.png 또는 jpg/jpeg/webp 선택
```

출력:

```text
output/production_plan.json
output/subtitles.ass
output/rendered_typography_video.mp4
output/lyric_video.mp4
output/logs/pipeline.log
output/logs/render.log
output/logs/ffmpeg.log
```

최종 MP4 조건:

```text
video codec: h264
audio codec: aac
pixel format: yuv420p
resolution: 기본 1920x1080
duration: production_plan.json의 project.duration_total과 1.5초 이내 일치
```

## 런타임 요구사항

- Node.js 20 이상
- FFmpeg와 ffprobe가 `PATH`에 있어야 함
- Windows 호환 필수
- 프로젝트는 ESM 사용: `package.json`에 `"type": "module"`
- 기본 렌더러는 Motion Canvas가 아니라 FFmpeg-only 렌더러
- Motion Canvas 관련 소스는 실험용으로 둘 수 있지만 기본 파이프라인을 막으면 안 됨

## 전체 파이프라인

`src/pipeline/runLyricVideoPipeline.js`는 다음 순서로 실행한다.

```text
1. input 파일 검증
2. npm run plan
3. npm run subtitles
4. npm run render:typography
5. npm run compose -- --audio <audioPath> --no-subtitles
6. ffprobe 기반 최종 검증
```

CLI 옵션:

```text
--clean
--debug
--audio <path>
--title <text>
--artist <text>
--skip-plan
--skip-subtitles
--skip-motion
--skip-compose
```

Windows에서 npm script 실행은 `spawn('cmd.exe', ['/d', '/s', '/c', 'npm', 'run', script, '--', ...args])` 방식을 사용하라. 이는 Windows의 `spawn EINVAL` 문제를 피하기 위함이다.

각 단계가 끝나면 해당 산출물이 존재하고 0바이트보다 큰지 확인하라. 실패 시 `output/logs/pipeline.log`에 기록하고 즉시 종료하라.

## inputDiscovery.js

`input/` 아래 파일을 자동 탐색한다.

규칙:

```text
lyrics:
  1. 정확히 lyrics.json 우선
  2. 그 외 .srt
  3. 그 외 .lrc
  4. 그 외 .json

audio:
  첫 번째 .mp3 또는 .wav

backgroundImage:
  첫 번째 .png, .jpg, .jpeg, .webp

backgroundVideo:
  선택 사항. .mp4, .mov, .webm, .avi, .mkv
```

내보낼 함수:

```js
discoverInputFiles(inputDir = 'input')
isAudioFile(filePath)
isLyricFile(filePath)
isBackgroundImageFile(filePath)
isBackgroundVideoFile(filePath)
```

## timecode.js

구현 함수:

```js
formatTimestamp(totalSeconds)
isValidTimestamp(value)
```

타임스탬프 형식:

```text
HH:MM:SS.mmm
```

`formatTimestamp`는 음수나 비정상 숫자를 받으면 오류를 던진다.

## lyricParsers.js

구현 함수:

```js
readLyricInput(filePath)
parseSrt(raw)
parseLrc(raw)
```

지원 형식:

- `.json`: 그대로 JSON parse
- `.srt`: SRT 블록을 `timed_segments`로 변환
- `.lrc`: LRC 타임태그를 `timed_segments`로 변환

반환 형태:

```js
{
  lyrics: "가사 전체 텍스트",
  timed_segments: [
    {
      start: "00:00:04.867",
      end: "00:00:06.702",
      duration: 1.835,
      lyric: "가사 한 줄"
    }
  ],
  source_format: "srt"
}
```

규칙:

- UTF-8 BOM 제거
- 빈 줄 무시
- `[Intro]`, `[Verse]`, `[Instrumental]`, `[End]`처럼 메타데이터만 있는 줄은 가사에서 제외
- SRT는 시작/끝 시간을 그대로 사용
- LRC는 다음 타임태그를 현재 줄의 끝으로 사용
- LRC 마지막 줄은 `+4s`를 끝으로 사용
- 잘못된 타임스탬프, 빈 가사, 끝 시간이 시작보다 빠른 구간은 제외

## validation.js

구현 함수:

```js
validatePlannerInput(input)
validateProductionPlan(plan)
```

검증:

- input은 객체여야 함
- `lyrics`는 비어 있지 않은 문자열이어야 함
- `aspect_ratio`는 `16:9`, `9:16`, `1:1`만 허용
- `duration_per_line`은 0보다 큰 숫자
- production plan의 segment id는 1부터 순차적
- segment start/end는 `HH:MM:SS.mmm` 형식

## generateProductionPlan.js

`output/production_plan.json`을 생성한다.

CLI:

```text
--input <path>
--output <path>
```

기본값:

```text
input: 자동 탐색된 lyric file
output: output/production_plan.json
```

중요 구현:

- `discoverInputFiles('input')`로 가사와 오디오를 찾는다.
- `readLyricInput`으로 SRT/LRC/JSON을 모두 처리한다.
- `ffprobe`로 오디오 길이를 읽는다.
- `project.duration_total`은 `max(마지막 가사 end, 오디오 길이)`로 저장한다.
- 오디오가 없거나 ffprobe 실패 시 마지막 가사 end를 사용한다.

기본 프로젝트 설정:

```js
aspect_ratio: '16:9'
resolution: '1920x1080'
fps: 30
mood: 'reflective'
genre: 'contemporary pop'
language: 'auto'
visual_style: 'original kinetic typography with soft abstract backgrounds'
```

지원 해상도:

```js
{
  '16:9': '1920x1080',
  '9:16': '1080x1920',
  '1:1': '1080x1080'
}
```

각 segment는 반드시 다음 필드를 포함한다.

```text
id
start
end
duration
lyric
scene_summary
image_prompt
negative_prompt
motion.camera
motion.background_motion
motion.text_animation
motion.transition
style.color_palette
style.lighting
style.background
style.emotional_tone
ffmpeg_notes
```

프롬프트와 문구는 저작권 캐릭터, 특정 아티스트/스튜디오 모방, 로고, 워터마크, 노골적/위험한 이미지를 피하도록 작성한다.

## generateAssSubtitles.js

`output/subtitles.ass`를 생성한다.

CLI:

```text
--input <path>
--output <path>
```

기본값:

```text
input: output/production_plan.json
output: output/subtitles.ass
```

ASS 파일 섹션:

```text
[Script Info]
[V4+ Styles]
[Events]
```

지원 aspect ratio별 스타일:

```text
16:9 -> PlayResX 1920, PlayResY 1080
9:16 -> PlayResX 1080, PlayResY 1920
1:1  -> PlayResX 1080, PlayResY 1080
```

검증:

- `segments`는 비어 있지 않은 배열
- 각 segment는 start, end, lyric 필요
- start/end 타임스탬프 유효
- end > start
- 이전 segment와 overlap 금지

자막 텍스트 escape:

- `\` escape
- `{`와 `}` escape
- 줄바꿈은 `\N`

`karaoke_highlight`가 true이면 단어별로 `{\k...}` 태그를 균등 분배한다.

## renderTypographyVideo.js

`output/rendered_typography_video.mp4`와 `output/logs/render.log`를 생성한다.

CLI:

```text
--plan <path>
--subtitles <path>
--out <path>
--log <path>
--background <path>
--background-video <path>
--no-motion
--motion-strength low|medium|high
```

기본값:

```text
plan: output/production_plan.json
subtitles: output/subtitles.ass
out: output/rendered_typography_video.mp4
log: output/logs/render.log
motion-strength: low
```

구현:

- `production_plan.json`에서 aspect ratio, fps, 마지막 segment end를 읽어 렌더 길이를 결정한다.
- 배경 이미지가 있으면 `-loop 1`로 읽고 scale/crop한다.
- 이미지 모션이 켜져 있으면 `zoompan`으로 천천히 확대/이동한다.
- 배경 영상이 있으면 `-stream_loop -1`로 읽고 scale/crop한다.
- 배경이 없으면 `nullsrc`와 `geq`로 추상 배경을 만든다.
- 모든 경로에서 readability overlay, vignette, 선택적 noise를 적용한다.
- `subtitles` 필터로 ASS 자막을 burn-in한다.
- 출력은 silent H.264 MP4이다.

FFmpeg 출력:

```text
-c:v libx264
-pix_fmt yuv420p
-movflags +faststart
```

`escapeFilterPath`는 Windows 경로를 `/`로 바꾸고 드라이브 콜론을 escape해야 한다.

## composeFinalVideo.js

최종 `output/lyric_video.mp4`와 `output/logs/ffmpeg.log`를 생성한다.

CLI:

```text
--audio <path>
--video <path>
--out <path>
--no-subtitles
--title <text>
--artist <text>
```

기본값:

```text
video: output/rendered_typography_video.mp4
audio: 자동 탐색된 input/*.mp3 또는 input/*.wav
out: output/lyric_video.mp4
```

중요 안정화 요구사항:

- 최종 duration은 `production_plan.json`의 `project.duration_total`을 우선 사용한다.
- plan duration이 없으면 audio duration을 사용한다.
- duration을 알 수 없으면 명확한 오류를 던진다.
- 절대 `[0:v]loop=loop=-1:size=1` 같은 무한 영상 스트림에 `-shortest`만 의존하지 말라.
- 최종 영상 스트림은 반드시 유한해야 한다.

필수 FFmpeg 필터 구조:

```text
[1:a]atrim=duration=<duration>,asetpts=PTS-STARTPTS,asplit=2[amwav][aout]
[amwav]aformat=channel_layouts=mono[am]
[am]showwaves=s=<width>x80:mode=p2p:colors=0xCCEEFF:rate=<fps>:scale=sqrt[wave]
[0:v]tpad=stop_mode=clone:stop_duration=<duration>,trim=duration=<duration>,setpts=PTS-STARTPTS[vbase]
[vbase]drawbox=... [vbg]
[vbg][wave]overlay=x=0:y=H-80[vwave]
```

이후 선택적으로:

```text
subtitles filter
top progress bar
title/artist intro overlay
```

최종 출력 옵션:

```text
-map [finalVideoLabel]
-map [aout]
-c:v libx264
-pix_fmt yuv420p
-c:a aac
-b:a 192k
-t <duration>
-movflags +faststart
```

이 구조는 FFmpeg가 정상 종료되도록 보장하며, 깨진 MP4의 대표 원인인 `moov atom not found`를 방지한다.

## mediaProbe.js

`ffprobe`로 최종 MP4를 검증한다.

구현 함수:

```js
validateFinalMediaOutput({mediaPath, planPath})
ffprobe(mediaPath)
```

검증:

- 파일 존재
- 0바이트 초과
- video stream 존재
- video codec은 `h264`
- resolution은 plan의 `project.resolution`과 일치
- audio stream 존재
- audio codec은 `aac`
- duration > 0
- plan의 `project.duration_total`과 최종 duration 차이가 1.5초 이하

실패 시 오류 메시지는 다음처럼 여러 항목을 보여준다.

```text
ffprobe validation failed:
- Final MP4 has no audio stream.
- Final MP4 resolution must be 1920x1080, got ...
```

## run.bat

Windows 실행용 배치 파일을 만든다.

```bat
@echo off
chcp 65001 >nul
cd /d %~dp0

echo ========================================
echo  Lyric Video Generator
echo ========================================
echo.

echo [CLEAN] output 폴더 초기화 중...
if exist output (
    rd /s /q output
)
mkdir output
mkdir output\logs
echo [CLEAN] 완료
echo.

echo [1/2] 파이프라인 실행 중...
call npm run lyric-video -- --clean
if %errorlevel% neq 0 (
    echo.
    echo [FAILED] 파이프라인 실패. 로그 확인: output\logs\pipeline.log
    exit /b 1
)

echo.
echo [2/2] 출력 검증 중...
call npm run validate:media
if %errorlevel% neq 0 (
    echo.
    echo [FAILED] 검증 실패.
    exit /b 1
)

echo.
echo ========================================
echo  완료. 출력 파일: output\lyric_video.mp4
echo ========================================
```

## 구현 메모

- `production_plan.json`의 `project.duration_total`은 오디오가 가사보다 긴 경우 오디오 길이를 반영해야 한다.
- `rendered_typography_video.mp4`는 가사 마지막 구간까지만 만들어져도 된다.
- `composeFinalVideo.js`가 최종 오디오 길이까지 마지막 프레임을 `tpad=stop_mode=clone`으로 연장한다.
- 마지막 가사 이후 오디오만 남는 구간은 영상 마지막 프레임, waveform, progress bar가 유지된다.
- 최종 duration 기준은 plan duration으로 통일한다.
