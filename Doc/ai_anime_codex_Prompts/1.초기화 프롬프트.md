# AI Anime MV System Analysis Prompt

아래 프롬프트는 이 프로젝트를 나중에 다시 분석하거나 개선할 때 다른 AI/개발자에게 그대로 전달하기 위한 문서입니다. 프로젝트 루트 기준으로 읽고, 실제 소스와 생성 산출물을 확인한 뒤 개선안을 제안하거나 구현하세요.

---

## 분석 요청 프롬프트

당신은 Python 기반 로컬 웹앱과 영상 제작 프롬프트 파이프라인을 분석하는 시니어 엔지니어입니다. 이 저장소는 노래 가사/메타데이터/LRC/SRT 파일을 입력받아 애니메이션 MV 제작용 캐릭터 모델시트, 씬별 이미지 프롬프트, 씬별 비디오 프롬프트, 전체 스토리 설명을 생성합니다.

목표는 다음입니다.

1. 전체 코드 구조와 데이터 흐름을 이해한다.
2. 캐릭터 일관성, 스토리 일관성, 이미지-비디오 제작 흐름이 올바른지 점검한다.
3. 웹 UI 사용성이 실제 제작 흐름과 잘 맞는지 확인한다.
4. 프롬프트 파일은 영상/이미지 생성 도구에 넣기 좋게 영어 중심으로 유지하고, 한글 설명은 별도 요약 파일/웹 섹션에만 표시한다.
5. 필요한 개선사항이 있으면 기존 구조를 크게 깨지 않고 구현한다.

## 핵심 제작 의도

### 1. 캐릭터 모델시트는 MV당 1개

현재 구조는 씬마다 모델시트를 만드는 방식이 아닙니다. 한 곡/MV당 먼저 공통 캐릭터 턴어라운드 모델시트 1개를 만듭니다.

파일:

```text
prompts/image_prompts/00_character_turnaround_model_sheet.md
character/character_reference_prompt.md
```

이 모델시트는 다음 정보를 포함하도록 프롬프트가 작성됩니다.

```text
- front full-body view
- left side full-body view
- right side full-body view
- back full-body view
- three-quarter full-body view
- face close-up
- prop close-up of the paper crane
```

### 2. 씬별 이미지는 모델시트를 참조해서 생성

각 씬 이미지 프롬프트는 공통 캐릭터 모델시트를 identity lock으로 첨부하라고 지시합니다.

예상 제작 순서:

```text
1. 00_character_turnaround_model_sheet.md로 공통 모델시트 이미지 생성
2. scene_01_intro.md 이미지 생성 시 모델시트 이미지를 참조로 첨부
3. scene_02_verse.md 이미지 생성 시 같은 모델시트를 참조로 첨부
4. 모든 씬 반복
```

### 3. 비디오 생성 시 이미지 2개 첨부 권장

비디오 생성 도구가 여러 참조 이미지를 지원하면 다음 2개를 같이 첨부하는 것이 가장 안전합니다.

```text
1차 참조: 해당 씬별 이미지
2차 참조: 전체 캐릭터 턴어라운드 모델시트
```

비디오 프롬프트는 이 흐름을 명시합니다.

```text
Use the attached scene image as the primary first-frame/image-to-video reference.
If the tool supports multiple references, also attach the character turnaround model sheet as a secondary identity reference.
```

## 주요 스크립트 역할

### scripts/common.py

공통 유틸리티입니다.

- `PROJECT_ROOT`
- `ensure_directories()`
- `read_text()` with UTF-8, UTF-8-SIG, CP949, EUC-KR fallback
- `write_json()`, `read_json()`, `write_text()`
- `slugify()`, `timestamp()`, `versioned_run_dir()`
- `split_csv()`, `normalize_section_name()`

### scripts/parser.py

입력 파일을 `song_master.json`으로 변환합니다.

지원 입력:

```text
.txt : 메타데이터, 스타일 태그, 섹션 가사
.lrc : timestamped lyrics
.srt : subtitle-style timestamped lyrics
```

주요 기능:

- `input` 폴더 전체 스캔
- `raw_song.txt` 우선 사용
- LRC/SRT timed lyrics 파싱
- `[Intro]`, `[Verse 1: ...]`, `[Pre-Chorus: ...]`, `[Post-Chorus: ...]` 섹션 인식
- metadata field 인식:
  - Title
  - Genre
  - BPM
  - Mood
  - Energy
  - Instruments
  - Music style tags
  - Negative tags
  - Visual cues
  - Atmosphere
  - Pacing
- LRC/SRT에 섹션 마커가 있으면 timed lyrics 기반으로 sections 재구성

주의할 점:

- 현재 파싱은 정규식 기반입니다.
- 복잡한 malformed LRC/SRT나 다국어 인코딩은 추가 테스트가 필요합니다.
- 가사와 스타일 태그가 섞인 Suno류 입력을 어느 정도 처리합니다.

### scripts/emotion_engine.py

`song_master.json`을 읽어 감정/상징/환경 분석을 만듭니다.

출력:

```text
analysis/emotion_analysis.json
```

특징:

- 사전 기반 감정 매핑
- lonely, nostalgic, hopeful, sad 등 제한된 감정 맵
- section별 emotion, lighting, environment, camera_emotion 생성
- tension_curve 생성

개선 후보:

- 더 많은 감정 키워드 지원
- 한국어 mood 키워드 매핑
- 가사 내용 기반 감정 변화 분석
- BPM/섹션 길이 기반 긴장도 보정

### scripts/scene_generator.py

프로젝트의 핵심 생성기입니다.

출력:

```text
analysis/visual_world.json
analysis/cinematic_style.json
character/protagonist_bible.json
character/character_prompt.md
character/character_reference_prompt.md
storyboard/story_arc.json
storyboard/story_summary.md
storyboard/scene_list.json
storyboard/storyboard_prompts.md
storyboard/camera_directions.md
```

주요 기능:

- `create_visual_world()`
- `create_protagonist()`
- `create_story_arc()`
- `apply_story_arc_to_scenes()`
- `generate_scenes()`
- `image_prompt()`
- `video_prompt()`
- `character_reference_prompt()`

중요 설계:

- `story_summary.md`는 한글 설명용입니다.
- 이미지/비디오 프롬프트는 영어 중심으로 유지합니다.
- `scene_list.json`에는 `character_model_sheet` 항목이 있으며, Step 00 모델시트 정보를 포함합니다.
- 씬별 프롬프트에는 영어 `story_beat_en`만 들어가야 합니다.
- 한글 `story_beat_ko`는 웹 UI/한글 요약용입니다.

검증할 점:

- 프롬프트 파일에 한글이 섞이지 않는지 확인:

```powershell

print([str(p) for p in paths if re.search(r'[가-힣]', p.read_text(encoding='utf-8'))])
```

출력:


현재 동작:

```text
...
```

플랫폼:

```text

Luma
```

각 파일은 같은 scene video prompt에 플랫폼별 note를 덧붙입니다.

전체 CLI 파이프라인입니다.

기본 입력:

```text
input/
```

실행:

```powershell
python scripts/run_pipeline.py --input input --snapshot
```

스냅샷 대상:

### scripts/web_app.py

Python 표준 라이브러리 기반 로컬 웹 UI입니다. 별도 Flask/FastAPI 의존성은 없습니다.

기능:

- 가사/메타 입력 textarea
- title 입력
- LRC/SRT 파일 업로드
- `/generate` POST로 임시 입력 폴더 생성
- parser/emotion/scene/image/video generator 순차 실행
- 결과 표시:
  - 전체 스토리
  - Step 00 Character Turnaround Model Sheet
  - 씬별 가사
  - 씬별 스토리 연결
  - 씬별 이미지 프롬프트
  - 씬별 비디오 프롬프트
- JSON/text endpoint:
  - `/api/song_master`
  - `/api/story_arc`
  - `/api/scene_list`
  - `/character-reference`
  - `/story-summary`

주의:

- `cgi.FieldStorage`는 Python 최신 버전에서 deprecated 흐름이므로 장기적으로 대체 필요.
- 현재 업로드 허용 확장자는 `.lrc`, `.srt`입니다.
- 최대 업로드 크기: 8MB
- 실제 이미지/비디오 파일 업로드 및 관리 기능은 아직 없습니다.

### run_web.bat

Windows 더블클릭 실행용입니다.

동작:

```text
1. 프로젝트 루트로 이동
2. python PATH 확인
3. 브라우저에서 http://127.0.0.1:8000/results 열기

```

주의:

- 이미 8000 포트에 서버가 떠 있으면 충돌할 수 있습니다.
- 포트 점유 자동 종료/대체 포트 탐색은 아직 없습니다.

## 분석 시 꼭 확인할 품질 기준

### 캐릭터 일관성

- `00_character_turnaround_model_sheet.md`가 실제 결과 목록에서 Step 00으로 보이는가?
- 씬별 image prompt가 모델시트를 identity lock으로 쓰라고 명시하는가?
- video prompt가 scene image를 primary, model sheet를 secondary reference로 명시하는가?
- protagonist의 hair/outfit/shoulder bag/paper crane/body proportions가 모든 씬에서 유지되는가?

### 스토리 일관성

- `story_arc.json`에 한글 로그라인, 줄거리, acts, continuity_rules가 있는가?
- `story_summary.md`에 사용자가 이해할 수 있는 한글 전체 정리가 있는가?
- 각 scene에 다음 필드가 있는가?

```text
story_stage
story_beat_ko
story_beat_en
continuity_from_previous_ko
continuity_to_next_ko
story_prompt_context
```

### 프롬프트 언어 정책

- 이미지/비디오 생성용 프롬프트 파일은 영어 중심이어야 한다.
- 한글 설명은 `story_summary.md`, 웹 UI의 전체 스토리/스토리 연결 섹션에만 있어야 한다.
- `storyboard/storyboard_prompts.md`도 생성 도구용 prompt bundle로 볼 수 있으므로 영어 중심이어야 한다.

### 웹 UI 사용성

- 사용자가 모델시트를 먼저 생성해야 한다는 사실이 분명히 보이는가?
- 씬 이미지와 비디오 프롬프트의 사용 순서가 명확한가?
- LRC/SRT 첨부 후 결과가 바로 보이는가?
- JSON/MD 결과를 쉽게 확인할 수 있는가?

## 알려진 개선 후보

1. Multipart 업로드 처리 교체
   - 현재 `web_app.py`는 Python 표준 라이브러리의 `cgi.FieldStorage`를 사용합니다.
   - 장기적으로 Python 버전 업그레이드를 고려해 대체 구현 또는 경량 웹 프레임워크 검토가 필요합니다.

2. HTML/CSS 분리
   - 현재 `web_app.py`에 HTML/CSS가 inline으로 큼
   - 장기적으로 `templates/`, `static/` 분리 고려

3. 감정/스토리 분석 고도화
   - 현재 사전 기반
   - 한국어 가사 의미 분석은 약함
   - OpenAI API 또는 로컬 LLM 연동 후보

4. 섹션 길이/타임코드 기반 씬 duration 생성
   - LRC/SRT timestamps로 각 scene duration 계산 가능
   - video prompt에 권장 clip length 추가 가능

5. 프롬프트 검증기 추가
   - 프롬프트 파일 한글 포함 여부 검사
   - 캐릭터 consistency 키워드 누락 검사
   - scene count와 prompt count 일치 검사

## 최근 해결된 이슈

1. 웹 입력에서 `Title:`이 앞에 붙어도 다음 스타일/BPM 라인을 찾아 BPM과 style tags를 추론하도록 개선했습니다.
2. 웹 UI 스냅샷도 CLI와 동일하게 `slugify(song["title"])`를 사용하도록 통일했습니다.
3. `image_prompt_generator.py`는 `prompts/image_prompts/*.md`를 정리한 뒤 새 프롬프트를 생성하도록 개선했습니다.
4. `run_web.bat`는 서버 시작 후 브라우저를 열도록 순서를 조정했습니다.

## 개선 요청 시 유지해야 할 원칙

- 기존 Python 표준 라이브러리 기반 구조를 우선 유지한다.
- 새 의존성은 꼭 필요할 때만 추가한다.
- 프롬프트 파일과 한글 설명 파일의 역할을 섞지 않는다.
- 생성 도구용 프롬프트는 영어 중심으로 유지한다.
- 사용자 설명/스토리 요약은 한글로 별도 제공한다.
- 캐릭터 일관성 흐름은 유지한다:

```text
00 모델시트 이미지
→ 씬별 이미지 생성 시 모델시트 첨부
→ 비디오 생성 시 씬 이미지 + 가능하면 모델시트 첨부
```

- 출력 파일 개수와 웹 UI 표시가 실제 제작 순서와 일치해야 한다.
