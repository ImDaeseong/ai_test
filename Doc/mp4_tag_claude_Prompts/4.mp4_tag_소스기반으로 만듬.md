## 🎯 기능 명세

### 핵심 기능

1. URL 입력 → yt-dlp 우선 시도
2. yt-dlp 실패 시 → Playwright로 페이지 동적 분석
3. 수집된 m3u8 / mp4 스트림 분류
4. ffmpeg로 최적 화질 다운로드
5. Streamlit Web UI에서 2단계(분석 → 다운로드) 분리 워크플로우

### Windows asyncio 호환 (필수)

```python
def run_async(coro):
    loop = asyncio.ProactorEventLoop()
    try:
        asyncio.set_event_loop(loop)
        return loop.run_until_complete(coro)
    finally:
        loop.close()
```

### 2단계 워크플로우 (반드시 분리)

**1단계 — 분석만 수행**

```
URL 입력창 + [🔍 분석] 버튼
버튼 클릭 시:
  st.spinner("페이지 분석 중...") 표시
  collect_media_urls(url) 호출
  결과를 st.session_state["streams"]에 저장
  ※ 다운로드는 절대 실행하지 않는다
[분석] 버튼 재클릭 시: 기존 streams 초기화 후 재분석
```

**2단계 — 목록 표시 + 개별 다운로드**

```
st.session_state["streams"]에 항목이 있으면 아래 목록 유지 표시:

번호 | 타입 | URL (앞 80자)          | 버튼
  1  | HLS  | https://...            | [⬇️ 다운로드]
  2  | MP4  | https://...            | [⬇️ 다운로드]

각 행의 [다운로드] 버튼은 해당 항목만 처리
버튼 클릭 시:
  st.progress() 로 실시간 ffmpeg 진행률 표시
  HLS → get_best_m3u8() → download_with_ffmpeg()
  MP4 → download_with_ffmpeg() 직접
  저장 경로: downloads/video_HHMMSS.mp4
  완료 후 st.success("✅ 저장 완료: {파일명}") 표시
  다른 항목 버튼에 영향 없음
```
