# =========================================================
# PRIMARY OBJECTIVE
# =========================================================

Create a Windows MFC dialog application that:

- reads game definitions from `GameConfig.json`
- detects whether a target game is installed on the current PC
- identifies the correct launcher (Steam / Epic Games / Netmarble)
- launches the game via the appropriate platform launcher

The application must:

- run with administrator privileges (UAC RequireAdministrator)
- never hard-code game-specific values in C++ source
- use `GameConfig.json` as the single source of truth for game configuration
- support adding new games by editing only `GameConfig.json`
- be fault tolerant when registry keys or files are missing
- use structured logging to a daily rotating log file
- never crash during detection or launch flows

# CONFIGURATION REQUIREMENTS
# =========================================================

`GameConfig.json` must be the only source of game-specific values.

Schema:

```json
{
  "games": [
    {
      "id": "kingsroad",
      "enabled": true,
      "gameName": "Game of Thrones: Kingsroad",
      "exeName": "GameOfThrones-Kingsroad.exe",
      "steam": {
        "appId": "3183280",
        "commonDirs": [
          "Game of Thrones_Kingsroad",
          "Game of Thrones Kingsroad"
        ]
      },
      "epic": {
        "manifestKeyword": "KingsroadGoT"
      },
      "netmarble": {
        "regSubKey": "SOFTWARE\\Netmarble Corp\\gotasiaA",
        "folder": "KingsroadGoT"
      }
    }
  ]
}
```

Rules:
- Only one game entry may have `"enabled": true` at a time.
- `ApplyFromJsonText()` processes only the current schema.
- Legacy keys (`RunGameConfig`, `GameInfo`, `RunGame`, `game`) are not supported.
- If `GameConfig.json` is missing or no game is enabled, detection runs with empty config.
- Do NOT fall back to hard-coded game values when config load fails.

File search: starting from the executable directory, traverse up to 6 parent levels.

# =========================================================
# DETECTION REQUIREMENTS
# =========================================================

Implement three platform detectors. Run in this fixed order:

```
CGameInstallSearch::SearchAll()
  1. CSteamLauncherSearch::Search()
  2. CEpicLauncherSearch::Search()
  3. CNetmarbleLauncherSearch::Search()
```

Each `Search()` must:
- initialize a fresh `CGameInstallInfo` before filling it
- return TRUE and append to `arrResult` if launcher path OR game EXE exists
- return FALSE without appending if neither is found

### Steam detection flow

1. Read `HKCU\SOFTWARE\Valve\Steam` → `SteamExe`
2. If missing, scan A–Z fixed/remote drives for:
   - `{Drive}:\Program Files (x86)\Steam\steam.exe`
   - `{Drive}:\Program Files\Steam\steam.exe`
   - `{Drive}:\Steam\steam.exe`
3. Derive Steam root from parent of `steam.exe`
4. Parse `<SteamRoot>\steamapps\appmanifest_{appId}.acf` → `installdir`, `buildid`
5. Also check `steam.commonDirs[]` entries under `<SteamRoot>\steamapps\common\`
6. Launch command: `steam://rungameid/{appId}`

### Epic Games detection flow

1. Read `HKCU\SOFTWARE\Epic Games\EOS` → `ModSdkCommand`
2. If missing, scan drives for:
   - `{Drive}:\Program Files (x86)\Epic Games\Launcher\Portal\Binaries\Win32\EpicGamesLauncher.exe`
   - `{Drive}:\Program Files\Epic Games\Launcher\Portal\Binaries\Win64\EpicGamesLauncher.exe`
   - `{Drive}:\Epic Games\Launcher\Portal\Binaries\Win64\EpicGamesLauncher.exe`
3. Scan `%ProgramData%\Epic\EpicGamesLauncher\Data\Manifests\*.item` for:
   - `"LaunchExecutable"` == `exeName` from config
   - OR file contains `epic.manifestKeyword`
4. Extract: `InstallLocation`, `AppVersionString`, `CatalogItemId`, `CatalogNamespace`, `AppName`
5. Launch command: `com.epicgames.launcher://apps/{CatalogNamespace}%3A{CatalogItemId}%3A{AppName}?action=launch&silent=true`

### Netmarble detection flow

1. Read `HKCU\SOFTWARE\Netmarble Corp` → `AppDrive`
2. Read `HKCU\{netmarble.regSubKey}` → `path`
3. Launcher path: `{AppDrive}\Netmarble Launcher.exe`
4. If missing, scan drives for:
   - `{Drive}:\Program Files (x86)\Netmarble\Netmarble Launcher\Netmarble Launcher.exe`
   - `{Drive}:\Program Files\Netmarble\Netmarble Launcher\Netmarble Launcher.exe`
   - `{Drive}:\Netmarble\Netmarble Launcher\Netmarble Launcher.exe`
5. Game EXE: `{path}\{exeName}` or drive scan under `netmarble.folder`
6. Launch command: empty string — `LaunchGame` uses `m_strLauncherPath` directly

# =========================================================
# LAUNCH REQUIREMENTS
# =========================================================

`CGameInstallSearchCommon` public static methods for launch:

```cpp
static BOOL LaunchGame(const CGameInstallInfo& info, BOOL bDirect = FALSE, DWORD* pdwProcessId = NULL);
static BOOL LaunchNetmarble(const CGameInstallInfo& info, DWORD* pdwProcessId = NULL);
```

Private static helper:
```cpp
static BOOL LaunchExeDirect(LPCTSTR lpszExePath, LPCTSTR lpszInstallPath, DWORD* pdwProcessId);
```

`LaunchGame` dispatch logic:
1. `bDirect = TRUE` → call `LaunchExeDirect` with `m_strExePath`
2. `m_eType == GAME_INSTALL_NETMARBLE` → call `LaunchNetmarble`
3. Otherwise → `ShellExecute` with `m_strLaunchCommand` (Steam/Epic protocol URL)

`LaunchNetmarble`:
- If `m_strLauncherPath` is not empty: `ShellExecute(m_strLauncherPath)`
- Else if `m_strExePath` exists: `ShellExecuteEx(SEE_MASK_NOCLOSEPROCESS, m_strExePath)` → PID via `GetProcessId(hProcess)`
- Else: return FALSE

Return FALSE when:
- `m_strLaunchCommand` is empty and `m_strLauncherPath` is empty
- `ShellExecute` return value ≤ 32
- `ShellExecuteEx` fails

Note: Steam and Epic do not return a game PID from the protocol URL launch path.
