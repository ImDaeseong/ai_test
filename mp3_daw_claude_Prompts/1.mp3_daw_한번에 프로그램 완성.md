🚀 Claude 전용 프롬프트 (Copy & Paste용)
제목: Suno AI 음원 처리용 로컬 하이브리드 DAW 시스템 구축 (Go + Python)

작업 지시: 지금부터 현재 디렉토리에서 작업을 수행하라. 설명은 생략하고 즉시 실행 가능한 코드 작성 및 아키텍처 구성을 완료하라.

🚨 절대 규칙 (반드시 준수)

언어 배분: 시스템 제어 및 웹 서버는 Go(Wails 또는 Gin), 오디오 처리 핵심 로직은 Python으로 작성.

외부 API 금지: OpenAI 등 유료 API 사용 금지. 로컬 오픈소스 라이브러리만 사용.

환경 최적화: Windows 10/11 CPU 환경에서 즉시 실행 가능해야 함.

데이터 관점: 오디오를 단순 파일이 아닌 '데이터(주파수 대역, LUFS, Stem)' 관점에서 처리.

🛠️ 시스템 아키텍처 및 구성 파일

main.go (또는 Wails 프로젝트): 로컬 웹 서버 구동, Python 프로세스 관리, 파일 시스템 감시(fsnotify).

engine.py: Pedalboard, Demucs, Librosa를 이용한 실제 오디오 처리 엔진.

requirements.txt & go.mod: 필요한 모든 의존성 포함.

🎯 최종 목표 (Flow)

감시: Suno AI에서 다운로드된 파일이 특정 폴더에 들어오면 Go가 즉시 감지.

분석: Python 엔진이 주파수 응답(FFT) 및 라우드니스(LUFS)를 분석하여 JSON 데이터 생성.

처리 (Stem Separation): Demucs를 호출하여 보컬, 드럼, 베이스, 기타 트랙으로 자동 분리.

엔지니어링 (Mastering): Pedalboard를 사용하여 저역대 컷, 컴프레서 적용, -14 LUFS 정규화 수행.

시각화 데이터: 웹 UI(HTML/JS)에서 Wavesurfer.js로 보여줄 파형 데이터와 분석 결과 반환.

🐍 engine.py 구현 지침

analyze_audio(file_path): Librosa를 사용하여 BPM, Key, 정밀 주파수 대역 분석.

separate_stems(file_path): 로컬에 설치된 Demucs 모델을 호출하여 트랙 분리.

apply_mastering(file_path):

LowpassFilter (15kHz), HighpassFilter (30Hz) 적용.

Compressor를 통한 다이내믹 레인지 최적화.

pyloudnorm을 이용한 유튜브 최적화(-14 LUFS) 타겟팅.

🐹 main.go 구현 지침

func watchFolder(): fsnotify를 사용하여 신규 .mp3/wav 파일 감지 시 Python 엔진 호출.

func runPythonEngine(): exec.Command를 통해 Python 스크립트 실행 및 결과값(JSON) 수신.

func startServer(): 로컬 8080 포트에서 오디오 데이터 및 분석 결과를 서빙하는 API 엔드포인트 구현.

🔧 추가 요구사항

FFmpeg 체크: shutil.which("ffmpeg")로 설치 여부 사전 확인 및 가이드 출력.

오류 처리: 파일 점유 문제(다운로드 중 처리 방지) 해결을 위한 재시도 로직 포함.

로그: 모든 단계에서 한글 로그 출력 (진행률 / 처리 결과 / DB 기록 상태).
