## 🏗️ 시스템 아키텍처

```
[Suno AI 다운로드 폴더 (./inbox)]
        │  fsnotify 감시
        ▼
[main.go — Go 제어 레이어]
  ├─ Gin 웹 서버 (포트 8080)
  ├─ Job 시스템 (queued / running / done / error)
  ├─ 파이프라인 워커 고루틴 × 2
  └─ exec.Command → engine.py 호출
        │  JSON 통신 (stdout)
        ▼
[engine.py — Python 처리 엔진]
  ├─ analyze  : BPM / Key / FFT 주파수 대역 / LUFS
  ├─ master   : EQ → 컴프레서 → LUFS 정규화
  ├─ separate : Demucs Stem 분리
  └─ pipeline : analyze + master 묶음
        │
        ▼
[./output/  결과물 저장]
[static/index.html — 브라우저 UI]
```

## 🐹 main.go 구현 명세

### 의존성 (go.mod)
```
github.com/gin-gonic/gin
github.com/fsnotify/fsnotify
```

### Job 시스템
- `JobStatus` 구조체: `ID`, `File`, `Operation`, `State`, `StartedAt`, `UpdatedAt`, `Result`, `Error`
- `State` 값: `"queued"` → `"running"` → `"done"` | `"error"`
- `sync.RWMutex`로 맵 보호, `sync/atomic`으로 ID 채번
- `runJobAsync()`: 고루틴에서 Python 호출 후 상태 자동 갱신

### 환경변수 (기본값 포함)
```
PYTHON_BIN = "python"
WATCH_DIR  = "./inbox"
OUT_DIR    = "./output"
```

### runPythonEngine()
- `python -u engine.py <command> [args...]` 실행
- 환경변수 `PYTHONUTF8=1`, `PYTHONIOENCODING=utf-8` 강제 주입
- stdout을 `io.MultiWriter`로 터미널과 버퍼에 동시 출력
- `extractLastJSON()`: stdout에서 마지막 유효한 JSON 객체(`{...}`) 추출

### 파일 감시 (watchFolder)
- `fsnotify`로 `./inbox` 감시
- `Create` / `Write` 이벤트에서 `.mp3 .wav .flac .m4a` 필터링
- 파일명에 `_mastered` 포함 시 **무시** (재처리 루프 방지)
- 쿨다운 3초 — 동일 파일 중복 이벤트 방지
- `workQueue chan string` 크기 32

### 파이프라인 워커
- 고루틴 2개 상시 대기
- `processFile()`: ① analyze → ② master (출력: `./output/<stem>_mastered.wav`)
- 동일 파일의 파이프라인이 이미 실행 중이면 건너뜀

### API 엔드포인트 (Gin, 포트 8080)

| Method | Path | 동작 |
|--------|------|------|
| `GET`  | `/api/jobs` | 전체 Job 목록 |
| `GET`  | `/api/job/:id` | 단일 Job 폴링 |
| `POST` | `/api/upload` | 파일 업로드 → `./inbox` 저장 |
| `POST` | `/api/analyze` | **동기** 분석 (수 초) |
| `POST` | `/api/master` | **비동기** 마스터링 → Job ID 반환 |
| `POST` | `/api/separate` | **비동기** Stem 분리 → Job ID 반환 |
| `POST` | `/api/pipeline` | 파이프라인 큐 등록 |
| `GET`  | `/api/check` | 의존성 확인 |
| `GET`  | `/audio/*` | `./output` 정적 서빙 |
| `GET`  | `/` | `./static/index.html` |

- CORS: `Access-Control-Allow-Origin: *`
- 시작 시 `./inbox`, `./output` 자동 생성
- 시작 2초 후 기존 inbox 파일 자동 재처리 예약

## 🐍 engine.py 구현 명세

### stdout UTF-8 강제 (최상단)
```python
if hasattr(sys.stdout, 'reconfigure'):
    sys.stdout.reconfigure(encoding='utf-8', errors='replace')
    sys.stderr.reconfigure(encoding='utf-8', errors='replace')
else:
    import io
    sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8',
                                  errors='replace', line_buffering=True)
    sys.stderr = io.TextIOWrapper(sys.stderr.buffer, encoding='utf-8',
                                  errors='replace', line_buffering=True)
```

### 로거 설정
- 포맷: `[HH:MM:SS] LEVEL │ 메시지`
- 한글 이모지 로그 (✅ ❌ ⚠️ 🔍 🎵 📊 등)
- `force=True`로 중복 핸들러 방지

### check_dependencies()
- `shutil.which("ffmpeg")` 로 FFmpeg 확인, 없으면 경고 + 설치 URL 안내
- `librosa`, `pyloudnorm`, `pedalboard`, `soundfile`, `demucs` import 체크
- 결과: `{"ok": bool, "missing": [...], "warnings": [...]}`

### load_audio_with_retry(file_path, max_retries=5, delay=2.0)
- 파일 크기가 **2회 연속 동일**할 때까지 대기 (다운로드 중 처리 방지)
- `librosa.load(sr=None, mono=False)` 로드
- 반환: `y (channels, samples)`, `y_mono (samples,)`, `sr`

### analyze_audio(file_path) → dict
```
librosa.beat.beat_track()          → BPM
librosa.feature.chroma_cqt()      → Key (C/C#/D … B)
librosa.stft() + fft_frequencies() → 7개 주파수 대역 에너지
pyloudnorm.Meter.integrated_loudness() → LUFS
_compute_waveform_peaks(y_mono, 1000)  → Wavesurfer.js용 피크 배열
```

**주파수 대역 7개:**
- `sub_bass_20_60hz`
- `bass_60_250hz`
- `low_mid_250_500hz`
- `mid_500_2khz`
- `upper_mid_2k_4khz`
- `presence_4k_8khz`
- `brilliance_8k_16khz`

**반환 키:** `file`, `duration_sec`, `sample_rate`, `bpm`, `key`,  
`lufs_integrated`, `frequency_bands`, `waveform_peaks`, `status`

### separate_stems(file_path, output_dir, model="htdemucs") → dict
- `subprocess.run([python, "-m", "demucs.separate", "--name", model, "--out", out_path, src])`
- 출력 경로: `{output_dir}/{model}/{stem_name}/{vocals|drums|bass|other}.wav`
- 최초 실행 시 모델 가중치 자동 다운로드 (~1.5 GB) 안내 로그 출력
- 기본 output_dir: `./output/stems` (inbox 재처리 루프 방지)

### apply_mastering(file_path, target_lufs=-14.0, output_path=None) → dict
```
soundfile.read()  →  (samples, channels) float64

pedalboard.Pedalboard([
    HighpassFilter(cutoff_frequency_hz=30.0),
    LowpassFilter(cutoff_frequency_hz=15000.0),
    Compressor(threshold_db=-18.0, ratio=3.0,
               attack_ms=10.0, release_ms=150.0),
])

pyloudnorm.normalize.loudness() → target LUFS 정규화
피크 클리핑 방지: peak > 0.989 이면 0.989/peak 스케일
soundfile.write(subtype="PCM_24")
```
**반환 키:** `file`, `output`, `original_lufs`, `final_lufs`, `target_lufs`, `sample_rate`, `status`

### CLI 서브커맨드 (Go exec.Command 호출용)
```
python engine.py check
python engine.py analyze <file>
python engine.py separate <file> [--output DIR] [--model MODEL]
python engine.py master <file> [--lufs FLOAT] [--output PATH]
python engine.py pipeline <file> [--lufs FLOAT]
```
- 모든 결과는 `json.dumps(..., ensure_ascii=False, indent=2)` 로 stdout 출력
- 예외 발생 시 `{"status":"error","message":"...","traceback":"..."}` 출력 후 `sys.exit(1)`

## 🖥️ static/index.html 구현 명세

### 기술 스택
- 순수 HTML + Vanilla JS (프레임워크 없음)
- `wavesurfer.js v7` CDN (`https://unpkg.com/wavesurfer.js@7/dist/wavesurfer.min.js`)
- 인라인 CSS (외부 파일 없음, 단일 파일 완결)

### 디자인 시스템
- 배경: `#0d0d0f`, 카드: `#18181b`, 테두리: `#27272a`
- 포인트 컬러: `#7c3aed` (보라), `#c4b5fd` (연보라)
- 폰트: `'Segoe UI', sans-serif`
- 반응형: 900px 이하에서 1열 그리드

### UI 구성 (6개 카드)

**① 파형 뷰어** (full-width)
- 드래그&드롭 / 클릭 업로드
- `WaveSurfer.create()` 로 파형 렌더링
- 재생 / 일시정지 / 정지 버튼, 시간 표시

**② 오디오 분석**
- 드래그&드롭 → `/api/upload` → `/api/analyze` (동기)
- 분석 완료 후 BPM·Key·LUFS·길이 stat-row 표시
- 주파수 대역별 바 차트 (최대값 대비 퍼센트)
- JSON 원문 result-box 표시

**③ 마스터링**
- 드래그&드롭 → `/api/upload` → `/api/master` (비동기)
- 타겟 LUFS 입력 필드 (기본값 `-14`)
- Job ID 수신 후 1.5초 간격 폴링 (`/api/job/:id`)
- 완료 시 `original_lufs → final_lufs` 표시

**④ Stem 분리**
- 드래그&드롭 → `/api/upload` → `/api/separate` (비동기)
- 모델 선택: `htdemucs` / `htdemucs_ft` / `mdx`
- 폴링 완료 시 vocals·drums·bass·other 경로 목록 표시

**⑤ 전체 작업 목록**
- `/api/jobs` 조회, 최신순 정렬
- 뱃지 색상: `queued`=회색, `running`=파랑, `done`=초록, `error`=빨강
- 수동 새로고침 + 2초 자동 갱신 토글

**⑥ 시스템 상태**
- `/api/check` 호출 → 의존성 JSON 표시

### Job 상태 UI 컴포넌트
- `.job-status` 패널: `queued` / `running` / `done` / `error` 클래스로 색상 전환
- `running` 상태: CSS 스피너 애니메이션 + 경과 시간 표시
- `setStatus(panel, state, msg)` 공통 함수

### 파일 업로드 플로우
```
onFileChosen(panel, file)
  → uploadFile() [POST /api/upload]
  → serverPaths[panel] = response.path
  → 버튼 활성화
```

## 🔧 오류 처리 및 품질 요구사항

| 항목 | 처리 방식 |
|------|-----------|
| 파일 점유 (다운로드 중) | `load_audio_with_retry()` 크기 안정화 대기 |
| inbox 재처리 루프 | `isAudioFile()`에서 `_mastered` 포함 파일 필터링 |
| 큐 포화 | `workQueue` 가득 찰 경우 503 반환 |
| 동일 파일 중복 파이프라인 | 실행 전 Job 맵 검사 후 건너뜀 |
| Python JSON 추출 실패 | `extractLastJSON()` 역방향 줄 탐색 |
| LUFS 측정 실패 | mono fallback 처리 |
| 피크 클리핑 | 정규화 후 `peak > 0.989` 자동 스케일 다운 |
