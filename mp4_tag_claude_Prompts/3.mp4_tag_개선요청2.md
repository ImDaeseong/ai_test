# 🔧 app.py 수정 지시

`app.py`만 수정한다. `main.py`, `requirements.txt`는 건드리지 말 것.

---

## ❌ 현재 문제

URL 입력 후 분석 버튼을 누르면 **즉시 다운로드까지 자동으로 실행**된다.

## ✅ 원하는 동작

**분석**과 **다운로드**를 완전히 분리된 2단계로 구현한다.

---

## 📋 2단계 워크플로우 (반드시 준수)

### 1단계: 분석만 수행
- URL 입력창 + [분석 시작] 버튼
- 버튼 클릭 시:
  - `try_ytdlp(url)` 시도
    - 성공 → `st.success("yt-dlp로 다운로드 완료")` 출력 후 **종료** (다운로드 목록 표시 안 함)
    - 실패 → `find_stream_with_playwright(url)` 호출
  - 발견된 스트림을 `st.session_state["streams"]`에 저장
  - **다운로드는 절대 하지 않는다**

### 2단계: 목록 표시 + 개별 다운로드
- `st.session_state["streams"]`에 항목이 있으면 아래 목록을 화면에 표시:

  | 번호 | 유형 | URL (앞 80자) | 버튼 |
  |------|------|--------------|------|
  | 1 | m3u8 | https://... | [다운로드] |
  | 2 | mp4  | https://... | [다운로드] |

- 각 행의 [다운로드] 버튼은 **해당 항목만** 다운로드
- 버튼 클릭 시:
  - `st.spinner("다운로드 중...")` 표시
  - m3u8 → `get_best_m3u8()` → `download_with_ffmpeg()`
  - mp4 → `download_with_ffmpeg()` 직접 호출
  - 저장 경로: `downloads/output_YYYYMMDD_HHMMSS.mp4`
  - 완료 후 `st.success("저장 완료: {파일명}")` 표시
  - **다른 항목의 [다운로드] 버튼은 영향받지 않는다**

---

## 🔑 session_state 구조

```python
st.session_state["streams"]  # list[dict] - 분석 결과 저장
# 예: [{"url": "...", "type": "m3u8", "headers": {...}}, ...]

st.session_state["download_status"]  # dict - 각 항목 다운로드 결과
# 예: {0: "완료: output_20250101_120000.mp4", 1: None}
```

---

## ⚠️ 주의사항

- [분석 시작] 버튼을 다시 누르면 기존 `st.session_state["streams"]` 초기화 후 재분석
- 목록은 분석 완료 후 화면에 계속 유지 (버튼 클릭해도 목록 사라지지 않음)
- 각 [다운로드] 버튼은 `key=f"dl_{i}"` 형태로 고유 key 부여
