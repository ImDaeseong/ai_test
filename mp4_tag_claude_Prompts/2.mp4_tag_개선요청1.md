# 🚀 작업 지시: 기존 main.py를 유지하면서 app.py (Web UI) 신규 생성

## 📌 작업 범위 명확화 (반드시 먼저 읽을 것)

- `main.py` → **절대 수정하지 말 것**, 현재 상태 그대로 유지
- `app.py` → **신규 생성** (없으면 생성, 있으면 전체 덮어쓰기)
- `requirements.txt` → **streamlit 항목만 추가**, 나머지 기존 항목 유지
- `downloads/` 폴더 → 없으면 자동 생성 (코드 내에서 처리)

---

## 🎯 목표

`main.py`에 구현된 아래 함수들을 **직접 import**하여,  
Streamlit 웹 UI에서 브라우저로 조작할 수 있는 `app.py`를 작성한다.

`main.py`에서 import할 함수:
- `try_ytdlp(url)`
- `find_stream_with_playwright(url)`
- `get_best_m3u8(m3u8_url, headers)`
- `download_with_ffmpeg(url, headers, output_name)`

---

## 🛠️ app.py 구현 내용

### UI 흐름

```
1. URL 입력창 표시
2. [분석 시작] 버튼 클릭
   → st.spinner로 "스트림 수집 중..." 표시
   → try_ytdlp(url) 먼저 시도
     → 성공: "yt-dlp로 다운로드 완료" 메시지 표시 후 종료
     → 실패: find_stream_with_playwright(url) 호출
3. 수집된 스트림 목록을 표로 출력
   → 컬럼: 번호 | 유형(m3u8/mp4) | URL 앞 60자 | [다운로드] 버튼
4. [다운로드] 버튼 클릭 시
   → m3u8인 경우: get_best_m3u8() → download_with_ffmpeg()
   → mp4인 경우: download_with_ffmpeg() 직접 호출
   → 저장 경로: downloads/output_YYYYMMDD_HHMMSS.mp4
   → 완료 후 st.success()로 파일명 출력
```

### 필수 기술 요구사항

- `st.session_state["streams"]`에 분석 결과 저장 (버튼 클릭 후 목록 유지)
- `st.session_state["ytdlp_done"]`으로 yt-dlp 성공 여부 관리
- User-Agent / Referer 헤더는 `find_stream_with_playwright()`가 반환한 값 그대로 사용
- BANDWIDTH 최고 화질 선택은 `get_best_m3u8()` 내부 로직 그대로 사용
- `downloads/` 폴더 없으면 `os.makedirs("downloads", exist_ok=True)`로 생성
- ffmpeg / playwright 미설치 시 `st.error()`로 안내 메시지 표시
- 다운로드 중 `st.spinner()` 표시
- Windows 환경: `subprocess.run(..., shell=False)` 준수
