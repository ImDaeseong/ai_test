# 배포 전 체크리스트

> 복사 위치: 프로젝트 루트 `/PRE_DEPLOY.md`
> 사용 시점: 모든 배포(첫 배포 포함) 직전
> 소요 시간: 5~10분
> 원칙 (Gene Kim): "배포는 이벤트가 아니라 루틴이어야 한다. 루틴이 되려면 체크리스트가 짧아야 한다"

---

## 보안

- [ ] `.env` 또는 비밀값 파일이 커밋에 포함되지 않았다
- [ ] `git log --oneline -5` 확인 - 민감정보 포함 커밋 없음
- [ ] 프로덕션 환경변수가 스테이징/개발 값과 다르게 설정됨
- [ ] 외부에 노출되면 안 되는 엔드포인트에 인증이 걸려 있다
- [ ] HTTPS 적용 확인 (HTTP로 민감정보 전송 없음)

---

## 코드 상태

- [ ] AI 코딩 결과물은 `AI_CODING_REVIEW.md` 체크 완료
- [ ] 로컬에서 정상 동작 확인
- [ ] 기존 기능이 신규 변경으로 망가지지 않음 (회귀 확인)
- [ ] 에러 로그 출력 확인 (개발 중 테스트 로그 제거)

---

## 운영 준비

- [ ] 헬스체크 엔드포인트 동작 확인
- [ ] 에러 알림 수신 경로 확인 (이메일, Slack 등)
- [ ] 외부 API 키/크리덴셜이 프로덕션용으로 교체됨
- [ ] 비용 발생 API에 사용량 알림 설정됨

---

## 롤백 준비

- [ ] 롤백 방법을 지금 바로 실행할 수 있다
- [ ] DB 마이그레이션이 있다면 롤백 스크립트가 준비됨
- [ ] 이전 버전 태그/커밋 해시를 알고 있다

```
롤백 명령어 또는 절차:
```

---

## 첫 배포 추가 항목

> 첫 번째 배포에만 적용

- [ ] `PROJECT_START.md` 완성 여부 확인
- [ ] `README.md`에 실행 방법, 환경변수 목록이 있다
- [ ] 다른 사람이 이 프로젝트를 이어받을 수 있는 최소 문서가 있다
- [ ] 모니터링/로그 확인 방법을 팀원이 알고 있다 (회사 프로젝트)

---

## 배포 후 즉시 확인 (5분)

```
배포 완료 시각:
헬스체크 응답:
에러 로그 이상 없음: Yes / No
주요 기능 동작 확인:
```

배포 후 이상 감지 시 즉시 롤백 기준:
```
에러율 임계값: (예: 5% 이상이면 롤백)
응답시간 임계값: (예: 평소 대비 3배 이상이면 롤백)
```
